# 对战组件说明

对战组件负责玩家之间的切磋、赌约决斗、请求接受拒绝和最近决斗记录。

## 命令

```text
切磋 对方名称
切磋[CQ:at,qq=对方ID]
接受切磋 发起人名称 / 切磋接受 发起人名称
拒绝切磋 发起人名称 / 切磋拒绝 发起人名称
赌约 源石数量 对方名称
赌约 源石数量[CQ:at,qq=对方ID]
决斗 对方名称
决斗 源石数量 对方名称
接受赌约 发起人名称 / 赌约接受 发起人名称
拒绝赌约 发起人名称 / 赌约拒绝 发起人名称
赌约记录 / 决斗记录 / 记录决斗
```

## 规则

- 涉及对方的命令可以写玩家名称，也可以使用 CQ/at；WS 层会把 CQ/at 转为内部 client_id。
- `切磋` 是无押注对战。
- `赌约` 必须带正整数源石数量和对方。
- `决斗` 不带金额时等同切磋，带金额时等同赌约。
- 赌约参数支持金额和对方互换顺序，但推荐写法是 `赌约 源石数量 对方名称`。
- 发起请求时，双方都必须处于空闲状态，不能正在探险或休息。
- 每个玩家同一时间只能有一个未处理对战请求。
- 对战请求 10 分钟后超时。
- 赌约发起时先冻结发起人的押注源石；接受时再扣接受人的押注。
- 如果接受人源石不足，赌约取消并退回发起人冻结源石。
- 胜者获得双方押注扣除 3% 手续费后的奖池。
- 接受成功后立即结算，并返回逐次出手的详细代码块日志。
- 战败后精神归零。
- 战斗日志最多保留 7 天；清理每天最多执行一次。

## 扩展

- 对战核心结算使用根目录 `combat_core.py`，不是二级包互相导入。
- 修改请求有效期：改 SQL 插入时的 `+10 minutes`。
- 调整赌约手续费：改 `DuelService._accept()` 里的费率。
- 调整详细日志：改 `DuelService._duel_log_block()` 和逐回合格式化函数。
