# 探险组件说明

探险负责 30 分钟预计算、自动战斗、自动用药、掉落预览和最终领取。真正发奖和扣损失只在 `结束探险` 时发生。

## 命令

```text
地点 / 位置
地点列表
探险 / 探索
探险 地点名 / 探索 地点名
探险状态 / 状态探险 / 探索状态
结束探险 / 探险结束 / 结束探索 / 探索结束
掉落记录 / 记录掉落
```

## 流程

```text
探险 -> 预计算最多 30 分钟战斗 -> 死亡或 30 分钟到点停止 -> 探险状态可提前查看 -> 30 分钟后结束探险结算
```

## 规则

- `探险` 不带地点时使用玩家当前位置；`探险 地点名` 会先把玩家位置切到该探险地点再开始。
- 只有探险地点可以开始探险，特殊收购点、回收点不能探险。
- 地点推荐等级只用于提示风险，不限制进入。
- 每 90 秒预计算一场遭遇，30 分钟最多 20 场。
- 怪物等级按探险地点的等级段生成，高等级地点不会拦你，但更容易死亡。
- 探险占用玩家本体；探险中不能挑战首领、虫洞、切磋或赌约决斗。
- 探险中仍可跑商，这是行商化身在外界行动，不改变本次探险预计算地点。
- 开始探险时会按玩家的 `自动用药` 开关决定是否预消耗纳戒恢复类物品。
- 自动用药只处理恢复类物品，不会消耗宝石、技能书、开孔器或洗髓液。
- 预计算会在玩家死亡或 30 分钟到点时停止；背包装不下下一件背包掉落时，也会提前停止预计算。
- 预计算停止后不会主动推送消息，玩家必须发送 `结束探险` 才能领取。
- `结束探险` 必须等开始后 30 分钟才能结算；提前发送只会提示还要等多久。
- `探险状态` 可以提前查看阶段、剩余时间、停止原因、战斗胜负、预计经验、掉落预览、自动用药和下一步操作。
- `结束探险` 返回完整流程日志，包含战斗明细、逐次出手、掉落、经验和最终状态。
- 探险战败后精神会归零；死亡只停止本次预计算，不删除角色。
- 背包掉落进入背包，占负重；恢复类、宝石、技能书进入纳戒，不占负重；武器在结算时发放。

## 扩展

- 新增探险地点：改 `exploration_locations` 初始化数据。
- 新增怪物：改 `MONSTER_DEFS`。
- 新增地点特产掉落：先补齐跑商特产，探险会按当前位置读取额外掉落。
- 调整节奏：改 `EXPLORE_MINUTES` 和 `ENCOUNTER_SECONDS`。
- 调整完整日志：改 `ExplorationService._claim_log_block()` 和逐回合格式化函数。
