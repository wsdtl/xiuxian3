## 二、通讯规则

WS 连接：

```text
/ws/bot/{client_id}
```

标准消息：

```json
{
  "code": 202,
  "type": "text",
  "message": "修仙信息",
  "request_id": "本次请求唯一 id"
}
```

返回格式：

```json
{
  "code": 202,
  "type": "text",
  "message": "回复文本",
  "request_id": "原 request_id"
}
```

通讯约定：

- 正常业务回复使用 `code=202`。
- 协议错误、通讯错误、未命中触发器使用 `code=404`。
- `type` 由业务自己决定，常用 `text` 和 `image`。
- 图片回复的 `message` 是纯 base64 字符串，不带 `data:image/...` 头。
- `request_id` 是客户端等待回复的凭证，请求和回复必须对应。
- 文本回复由 `send_reply` 自动加 `【玩家名】` 前缀，图片回复不额外加标题。

命令拆分：

- 精确命令取第一个空格前文本为 `cmd`。
- 第一个空格后的原文作为 `message`，不主动改成列表。
- 没有空格时，WS 层会检查是否有紧贴命令的 CQ/at。
- 例如 `切磋[CQ:at,qq=xxx]` 会拆成命令 `切磋`，参数 `xxx`。
- 例如 `赌约 1000[CQ:at,qq=xxx]` 会把 CQ/at 转成 `xxx`，业务层仍按文本参数解析。

玩家引用：

- 当前正式指令优先使用玩家名称，因为新接口 id 很长。
- 名称唯一，避免对战、购买、接受请求时混淆。
- 涉及对方的命令也支持 CQ/at，WS 层会把 `[CQ:at,qq=xxx]` 统一转成内部 id。

## 三、玩家核心

创建规则：

- 命令：`创建用户 名称` / `用户创建 名称`。
- 名称 2 到 12 个字符。
- 名称不能包含空白字符。
- 名称全服唯一。
- 创建后默认 1 级、0 经验、凡体、空闲状态、天枢城位置。
- 创建时自动初始化源库、固定装备位。

改名规则：

- 命令：`改名 新名称`。
- 名称规则和创建一致。
- 改名冷却 24 小时。
- 名称只用于展示和指令查找，不改变玩家内部 id。

基础常量：

```text
每日刷新：04:00
等级上限：100
默认地点：天枢城
默认背包格子：80
默认负重：500
休息时间：1 分钟
玩家基础攻击常量：5
战斗记录保留：7 天
```

基础公式：

```text
1-40级升级经验 = floor(120 * level^1.85 + 600 * level)
40级后升级经验 = 基础经验 * (1 + 8*p^1.35 + 12*p^2.4 + 18*p^4.0)，p=(level-40)/59
血气上限 = 100 + 等级 * 12 + 体质值 * 8 + 装备血气加成
精神上限 = 60 + 等级 * 6 + 装备精神加成
人物基础攻击 = 5 + floor(等级 / 10)
防御 = 等级 + 体质值 * 2 + 装备防御加成
签到源石 = min(500 + 等级 * 30, 5000)
```

等级体验目标：

- 40 级前较顺滑，方便新玩家建立正反馈。
- 40 级后明显加难。
- 60 级后开始拉开升级差距。
- 80 级后进入长期区，专门压住每日十几次探险的高频玩家。
- 按满胜探险次数估算，1 到 100 级约需要 9000 次完美探险。
- 高等级刷低级怪经验衰减，低等级越级打怪经验提高。

体质系统：

- 默认体质为凡体。
- 洗髓液可重骰体质。
- 高级体质更强调特色，不做万金油。
- 体质效果会进入 `CoreService.equipment_bonuses()`，和固定装备、宝石一起汇总。

体质方向：

```text
均衡、身法、体修、恢复、防御、精神、木灵、冰脉、火阳、月华、剑骨、星命、山岳、仙肌、雷法、龙脉、金身、虚空
```

玩家命令：

```text
帮助
创建用户 名称 / 用户创建 名称
改名 新名称
修仙信息 / 状态
自动用药
自动用药 开启 / 自动用药 关闭
签到
新手礼包 / 礼包
休息
结束休息 / 休息结束
```

自动用药：

- 不带参数时查看当前状态。
- 开启词：开启、打开、启用、开、on、ON、1。
- 关闭词：关闭、关掉、停用、关、off、OFF、0。
- 只在探险预计算时生效。
- 只会消耗纳戒里的恢复类物品。

新手礼包：

```text
源石 10000
血契丹 2
阴冥草 2
```

休息：

- 玩家必须空闲。
- `休息` 后进入休息中。
- 1 分钟后发送 `结束休息` 或 `休息结束`。
- 结束休息恢复满血，并恢复一部分精神。
- 回春类宝石和体质会提高恢复效果。

## 四、库存系统

背包：

- 命令：`背包`、`使用 物品名`。
- 保存占负重物品。
- 跑商商品、怪物战利品、普通掉落进入背包。
- 背包检查格子上限、负重上限、堆叠上限。
- `使用` 只允许直接消耗恢复类物品。

纳戒：

- 命令：`纳戒`、`洗髓`。
- 保存不占负重物品。
- 恢复类、福袋、洗髓液、开孔器、宝石、技能书都进入纳戒。
- 宝石按名称和等级单独存储。
- `洗髓` 消耗洗髓液。

武器库：

- 武器独立存放在 `player_weapons`。
- 同名武器用实例 ID 区分。
- 展示格式会带出品质、等级、攻击、附魔数量。

固定装备：

- 创建玩家时默认拥有。
- 固定装备不进入背包或纳戒。
- 装备位：头部、左手、右手、左脚、右脚、饰品、护甲。

物品定义查询：

- 命令：`查看修仙物品 物品名`。
- 可查背包物品、纳戒物品、武器模板、武器自带技能、技能书附魔、体质资料。
- 查的是定义，不是玩家库存。
- 武器实例详情用 `查看武器 武器ID`。

## 五、物品库

背包物品：

- 代码中直接定义 33 种怪物战利品。
- 跑商特产 108 种会在数据库初始化时补入 `item_defs`。
- 背包物品有分类、品级、重量、堆叠上限、跑商状态、可使用状态、基准价、效果和说明。

纳戒物品：

- 恢复类：福袋、血契丹、阴冥草、回春露、凝神露、生骨丹、养魂丹。
- 消耗品：开孔器、洗髓液。
- 技能书：17 种，对应 17 种武器附魔效果。
- 宝石：9 种，按等级存储。

恢复类效果：

```text
福袋：按玩家等级段随机获得源石
血契丹：恢复 25% 血气
阴冥草：恢复 25% 精神
回春露：恢复 45% 血气
凝神露：恢复 45% 精神
生骨丹：恢复 70% 血气
养魂丹：恢复 70% 精神
```

特殊材料：

- 开孔器：固定装备开孔材料，只通过活动 Boss 掉落。
- 洗髓液：体质洗髓材料，只通过活动 Boss 掉落。
- 铭刻之羽：只通过岁时情劫首领产出，使用后删除。

## 六、经济系统

货币：

- 统一货币为源石。
- 随身源石可直接消费。
- 源库源石需要取出后才能消费。

主要收入：

- 签到。
- 探险经验和掉落。
- 普通跑商利润。
- 特殊战利品出售。
- 二手市场出售。
- 武器、宝石、技能书回收。
- 虫洞和首领奖励。
- 福袋。

主要支出：

- 源库升级。
- 跑商买货和手续费。
- 二手市场购买。
- 固定装备升级。
- 武器升级。
- 赌约押注。

经济保护：

- 普通跑商原地 4 小时转售限制。
- 普通跑商出售价格按最近买入价封顶，最高参考利润 45%。
- 跑商买入手续费 1%，出售手续费 2%。
- 二手市场手续费 5%。
- 特殊出售、武器回收、宝石回收、技能书回收都使用柔性降价曲线。
- 柔性降价不禁止玩家继续赚钱，只降低当天后续收益倍率。

## 八、二手市场

命令：

```text
二手市场 / 二手
二手市场上架 名称 数量 总价 / 上架二手市场 名称 数量 总价 / 二手上架 名称 数量 总价

二手市场购买 卖家名称 / 购买二手市场 卖家名称 / 二手购买 卖家名称
```

规则：

- 每个卖家同一时间只能上架一包商品。
- 可上架背包物品、纳戒物品、宝石和备用武器。
- 宝石按名称和等级区分。
- 武器按实例 ID 区分。
- 已装备武器不能上架。
- 最后一把武器不能上架。
- 武器上架会临时托管到市场 owner，购买后转移给买家。
- 买家支付总价，卖家收到扣除 5% 手续费后的源石。
- 购买时用卖家名称定位，也支持 CQ/at。

## 十、探险系统

命令：

```text

结束探险 / 探险结束 / 结束探索 / 探索结束

```

流程：

```text
探险 -> 预计算战斗和掉落 -> 死亡或 30 分钟到点停止 -> 探险状态可提前看 -> 30 分钟后结束探险结算
```

规则：

- `探险` 不带地点时使用当前位置。
- `探险 地点名` 会先切换到该探险地点再开始。
- 探险地点推荐等级只提示风险，不限制进入。
- 探险占用本体，探险中不能挑战首领、虫洞、切磋、赌约决斗。
- 探险中仍可跑商，解释为行商化身行动。
- 探险时长固定 30 分钟。
- 出怪间隔 90 秒。
- 最多预计算 20 场遭遇。
- 玩家死亡或 30 分钟到点时停止预计算。
- 背包装不下下一件背包掉落时也会提前停止预计算。
- 结束探险必须等 30 分钟后才能结算。
- `探险状态` 可查看当前预计算摘要。
- `结束探险` 返回完整日志，包含逐次出手、技能、伤害、掉落、经验、最终状态。
- 战败后精神归零。

掉落结构：

- 背包掉落：怪物战利品、地点特产。
- 纳戒掉落：恢复类、宝石、技能书。
- 武器掉落：先记录预览，结算时发放实例武器。

怪物经验：

```text
基础经验 = 80 + 怪物等级 * 22
平级倍率 = 1.0
怪物等级高于玩家：每级 +10%，最高 2.0
怪物等级低于玩家：每级 -8%，最低 0.15
最终经验 = 基础经验 * 等级差倍率 * 怪物系数
```

## 十一、战斗系统

战斗原则：

- 攻击力主要来自武器。
- 玩家本体只提供低基础攻击。
- 固定装备只提供血气、精神、防御等生存属性，不提供攻击力。
- 武器自带技能自动释放，不需要单独技能系统。
- 技能书附魔改变武器战斗效果。

伤害公式：

```text
有效防御 = 防御 * (1 - 穿透率)
固定减伤 = min(有效防御 * 0.2, 原始伤害 * 0.5)
中间伤害 = 原始伤害 - 固定减伤
比例减伤 = 有效防御 / (有效防御 + 300 + 30 * sqrt(有效防御))
比例减伤最高 60%
最终伤害 = max(1, int(中间伤害 * (1 - 比例减伤)))
```

战斗日志：

- 探险、虫洞、首领、切磋、赌约都返回逐次出手记录。
- 战斗记录最多保留 7 天。
- 清理每天最多执行一次。
