# 修仙落地设定

版本：2026-05-21

本文档是 `src/修仙` 模块的正式落地设定。目标是可以直接据此拆组件、建表、写服务和接入 WS 指令。

## 一、核心规则

- 玩家唯一身份是 `client_id`。
- 玩家需要先创建用户，创建时自定义展示名称。
- 展示名称只用于界面好看，不参与查找、交易、挑战和权限判断。
- 命令里需要指定玩家时，可以直接输入 `client_id`，也可以使用 `CQ/at`。
- 业务层只接收解析后的 `client_id`，不直接处理 `CQ/at` 文本。
- 模块内不使用群号、平台号、额外用户对象。
- 成长只使用 `等级 / 经验`。
- 等级上限为 100 级，按三年长期体验设计。
- 所有每日刷新、签到、每日统计、跑商奖励以每日 04:00 作为新一天边界。
- 业务日统一用 `当前时间 - 4 小时` 后的日期计算。
- 不做修炼系统，主要成长来自探险和打怪。
- 移动不耗时，导航后立即到达。
- 探险使用 30 分钟预计算，结果不主动推送。
- 用户发送 `结束探险` 后才展示结果、发放收益、扣除损失。
- 物品库进背包，占负重。
- 背包同时受格子、负重和物品堆叠上限限制。
- 装备库进纳戒，不占负重。
- 武器单独列库，不进背包和纳戒。
- 跑商商品只能来自物品库，并且全部占负重。
- 恢复类、宝石、技能书属于装备库，不参与跑商。
- 宝石用于固定装备镶嵌，旧称“镶嵌宝”不再使用。
- 武器是主要攻击来源，自带技能，可升级，可附魔，不能镶嵌宝石。
- 掉落武器拥有随机等级上限，等级上限决定长期养成空间。
- 武器附魔栏只由武器等级上限和当前等级决定，和武器品级独立。
- 固定装备默认拥有，可升级，可镶嵌，不提供攻击力。
- 金融系统按长期服设计，源石产出慢于长期养成消耗。

业务日示例：

```text
2026-05-21 03:59 仍属于 2026-05-20
2026-05-21 04:00 开始属于 2026-05-21
```

## 二、通讯和命令

### WS 输入

玩家发送：

```json
{
  "code": 202,
  "type": "text",
  "message": "存入源石 100",
  "request_id": "本次请求 id"
}
```

WS 驱动按第一个空格拆命令：

```text
cmd = 存入源石
message = 100
```

业务函数只认：

```text
client_id
message
match
```

`message` 是触发命令后的剩余文本。正则命令需要额外使用 `match`。

### 玩家引用

涉及对方玩家的命令，输入层支持两种写法：

```text
1684193123
[CQ:at,qq=1684193123]
```

解析规则：

- 直接输入 id 时，原样作为对方 `client_id`。
- 输入 `CQ/at` 时，只提取里面的 `qq` 值作为对方 `client_id`。
- 解析完成后再进入业务逻辑。
- 业务服务、数据库和战斗结算都只认 `client_id`。

### 返回格式

所有业务回复使用：

```json
{
  "code": 202,
  "type": "text",
  "message": "展示文本"
}
```

通讯错误、协议错误、未命中触发器使用 `404`。

业务提示、操作太快、服务繁忙、触发器超时使用 `202 text`。

### 创建用户

玩家第一次进入修仙系统前，需要先创建用户。

```text
创建用户 名称
```

规则：

- `client_id` 由 WS 驱动器传入，作为玩家唯一 id。
- `名称` 写入 `display_name`，只用于消息展示。
- 展示名称允许重复，因为真正查找玩家只认 `client_id`。
- 展示名称建议 2 到 12 个字符，超出时提示重新输入。
- 已创建过的玩家再次创建时，只提示已存在，不覆盖原数据。
- 除 `创建用户` 外，其他修仙命令发现玩家不存在时，统一提示先创建用户。

示例：

```text
创建用户 青衫客
```

### 改名

改名只修改展示名称，不影响玩家身份。

```text
改名 新名称
```

规则：

- 只能修改自己的 `display_name`。
- `client_id` 永远不变。
- 新名称建议 2 到 12 个字符。
- 名称允许重复。
- 24 小时内只能改名 1 次。
- 改名只影响之后的展示文本，历史记录不回写。

示例：

```text
改名 云游客
```

后续涉及其他玩家的命令可以用 id，也可以用 `CQ/at`：

```text
切磋 1684193123
切磋 [CQ:at,qq=1684193123]
赌约 1684193123 5000
赌约 [CQ:at,qq=1684193123] 5000
```

### 基础数值公式

```text
血气上限 = 100 + 等级 × 12 + 体质 × 8 + 固定装备、宝石、体质特性加成
精神上限 = 60 + 等级 × 6 + 固定装备、宝石、体质特性加成
人物基础攻击 = 5 + floor(等级 / 10)
防御 = 等级 + 体质 × 2 + 固定装备、宝石、体质特性加成
```

人物基础数值只提供生存和兜底输出。主要攻击来自武器。

### 体质系统

体质用于表现玩家根骨差异。体质名字用于展示，`physique` 数值用于计算血气和防御，`effect` 用于体现专属特性。

高阶体质不做万金油。越高级越强调路线：

- 守御路线：防御和承伤强，精神、探险或跑商笨重。
- 恢复路线：休息恢复和精神强，正面防御或血气偏弱。
- 探险路线：探险和闪避强，血防薄。
- 经济路线：跑商和寻机强，战斗身板薄。
- 血气路线：血厚，但精神、恢复或经营能力差。

洗髓液：

- 洗髓液属于纳戒消耗品，只通过活动 Boss 掉落，不进入普通探险掉落池。
- 命令：`洗髓`。
- `使用 洗髓液` 和 `使用装备库 洗髓液` 不会消耗洗髓液，只会提示改用 `洗髓`。
- 使用后按当前体质等级结算：低阶有较明显提升希望，中高阶主要同阶变化，圣阶不再继续上升。
- 单次洗髓最多向上跳 2 个体质等级，也可能小幅回落。
- 概率参考：凡阶/灵阶约 35% 提升、55% 同阶、10% 回落；玄阶约 22% 提升；地阶约 14% 提升；天阶约 7% 提升；圣阶 0% 提升。
- 洗髓后立即重算血气上限和防御；当前血气只会被新上限限制，不会自动回满。

### 等级曲线

```text
1 到 40 级：
升到下一级所需经验 = floor(120 × 当前等级 ^ 1.85 + 600 × 当前等级)

40 级后：
基础经验继续沿用原曲线，再按 40 到 99 的进度平滑加难。
最高到 99 级时约比原曲线多 40%。
```

| 当前等级 | 升到下一级所需经验 |
|---|---:|
| 1 | 720 |
| 10 | 14495 |
| 40 | 134405 |
| 50 | 205392 |
| 70 | 413503 |
| 90 | 727444 |
| 99 | 909643 |

1 到 100 级总需求约 `2876 万经验`。

### 怪物经验权重

怪物基础经验：

```text
基础经验 = 80 + 怪物等级 × 22
```

等级差经验倍率：

| 情况 | 倍率 |
|---|---:|
| 玩家等级 = 怪物等级 | 1.0 |
| 怪物高于玩家 | 每高 1 级 +10%，最高 2.0 |
| 怪物低于玩家 | 每低 1 级 -8%，最低 0.15 |

结算经验：

```text
怪物经验 = 基础经验 × 等级差倍率 × 胜负倍率
```

胜利倍率为 `1.0`，失败倍率为 `0.25`。

按每次探险 30 分钟、平均 4 场同级普通战斗估算：

| 活跃度 | 满级周期 |
|---|---|
| 每天 1 次探险 | 5 年以上 |
| 每天 2 次探险 | 约 3 年 |
| 每天 3 到 4 次探险 | 约 1.5 到 2.5 年 |
| 高频探险 + 精英 + 首领 | 约 1 到 2 年 |

```text

```

### 新手礼包

| 内容 | 数量 |
|---|---:|
| 源石 | 10000 |
| 血契丹 | 2 |
| 阴冥草 | 2 |

新手礼包只允许领取一次。

### 休息

- 只能在空闲状态使用。
- 发送 `休息` 后进入 `休息中`。
- 休息固定需要 1 分钟。
- 休息未满 1 分钟时，发送 `结束休息` 或 `休息结束` 只提示剩余时间。
- 休息满 1 分钟后，发送 `结束休息` 或 `休息结束` 才真正恢复并回到空闲。
- 结束休息时恢复血气。
- 结束休息时少量恢复精神。
- 不产出经验、源石和物品。

## 六、金融系统

### 源石定位

源石是唯一货币。

每日结算边界：

```text
每日 04:00 进入新的业务日
```

签到、源库单日利息、商场价格、商场热度和限流统计都按这个时间重置。

源石产出：

| 来源 | 定位 | 控制 |
|---|---|---|
| 签到 | 低保 | 每日小额 |
| 新手礼包 | 开局 | 一次性 |

| 福袋 | 小额随机补充 | 纳戒恢复类物品，随机源石 |
| 物品效果 | 随机惊喜 | 小概率 |

### 通胀检查

```text
系统新增源石 = 签到 + 新手礼包 + 跑商出售净收入 + 源库利息 + 福袋
系统回收源石 = 源库升级 + 固定装备升级 + 武器升级 + 宝石升级 + 二手市场手续费 + 商场手续费 + 赌约手续费
净增源石 = 系统新增源石 - 系统回收源石
```

连续多周净增过高时，下调跑商价格空间、特殊收购倍率、福袋金额或源库利率。

玩家普遍买不起基础升级时，上调前期跑商利润、签到低保或低级战利品价格。

- 每个玩家同一时间只能上架一包商品。
- 可以上架背包物品、纳戒物品或武器。

- 纳戒物品成交后进入买家纳戒。

- 玩家不能上架自己的最后一把武器。
- 武器上架后进入市场托管，卖家不能再切换、升级或重复出售这把武器。
- 武器下架后回到卖家武器栏，默认不自动装备。
- 不能买自己的商品。
- 上架免费。
- 成交收取 5% 手续费。
- 手续费从卖家成交款扣除。

示例：

```text
成交价 10000
买家支付 10000
系统回收 500
卖家获得 9500
```

特殊收购价格：

特殊收购是探险战利品的高价变现入口。

规则：

- 玩家需要先 `导航` 到对应特殊收购地点。
- 只有该地点配置的战利品可以特殊出售。
- 特殊出售不走商场买卖热度，不占用普通跑商利润公式。
- 特殊出售没有手续费，收入按 `物品基础价 × 地点倍率 × 数量` 计算。
- 恢复类、宝石、技能书、武器、跑商特产不进入特殊收购。
- 特殊收购不做硬额度限制，玩家一直可以出售战利品赚钱。
- 当天特殊出售收入越高，后续特殊收购价越低。
- 价格倍率公式：`1 / (1 + 今日收入 / 参考线 × 0.45)`，最低保留 35%。
- 参考线按等级提升：`50000 + 等级 × 15000`。
- 本次出售按 `今日收入 + 本次原价的一半` 估算倍率，避免一次性大批量出售完全绕过降价曲线。


品质系数沿用武器升级系数：凡品 1.0、良品 1.4、珍品 2.0、稀品 3.0。

### 商场价格

```text
市场价 = 基础价 × 供给因子 × 地点需求因子 × 每日波动
买入价 = 市场价 × 买入价差 × 买入热度因子
出售价 = 市场价 × 出售价差 × 出售热度因子
```

- 每个地点特产都必须写入显式定价，缺少定价会在数据库初始化时报错。
- 每个特产都配置：货物类型、品级、重量、堆叠、基础价、说明。
- 基础价是货物本身价值，不再使用名称哈希生成默认价格。
- 产地供给充足，所以本地更便宜；离产地越远，运输价越高。
- 不同地点有不同需求偏好，比如玄铁岭偏矿材、万药谷偏药材、玉京台偏宫货。
- 每天 04:00 进入新的业务日，价格种子随业务日变化。
- 每日波动范围是 0.95 到 1.07，只做轻微扰动。
- 产地供给因子是 `0.82`。
- 异地供给因子是 `0.96 + min(0.36, 距离 / 2600)`。
- 地点需求因子由 `TRADE_LOCATION_DEMANDS` 控制，普通货物默认 `0.98`。
- 买入价差是 `1.04`，买入热度因子是 `min(1.5, 1.04 + 当日买入数 × 0.01)`。
- 出售价差是 `0.82`。
- 出售热度因子是 `max(0.65, 1 - 当日出售数 × 0.008)`。

跑商利润由价格公式自然产生，不再单独写固定利润档位。

- 普通跑商出售会读取最近买入价，单件最高出售参考价不超过最近买入价的 145%。
- 商场推荐按 `单件净利润 / 物品重量` 排序，更适合有限负重下选货。
- 特殊收购只收对应怪物战利品，不受普通跑商利润上限影响。
- 当前代码暂未限制商场购买物品上架二手市场。
- 当前代码暂未实现每日跑商额外奖励。

## 十、探险系统

探险是经验、物品、装备库物品、武器和技能书的主要来源。

### 探险流程

```text
探险 -> 30 分钟预计算 -> 遇怪 / 掉落 -> 死亡、背包满或到点停止预计算 -> 等待结束探险
```

- 每个探险地点有 `推荐等级`，只用于提示风险。
- 只有探险地点可以开始探险，特殊收购地点不能探险。
- 玩家等级低于推荐等级时仍可进入，战斗风险由实际预计算结果承担。
- `地点列表` 会显示推荐等级和怪物等级段，方便玩家选择。
- 探险怪物按当前地点的怪物等级段生成，不再按玩家等级限制入口。
- `30 分钟到点` 只是预计算结束。
- `人物死亡` 只是预计算结束。
- `背包或负重满` 也是预计算结束。
- 不主动展示结果。
- 不主动发消息。
- 玩家发送 `探险状态` 可以提前看到预计算状态。
- 玩家发送 `结束探险` 才展示和领取。
- `结束探险` 必须等 30 分钟冷却到点后才能结算。
- 多场战斗连续继承上一场剩余血气和精神。
- 自动用药开启时，探险预计算会按阈值自动消耗纳戒里的恢复类药物。
- 自动用药只会使用恢复血气或精神的物品，不会自动打开福袋。
- 血气低于 45% 时尝试补到约 75%，精神低于 30% 时尝试补到约 65%。
- 自动用药消耗会在开始探险时扣除，结束探险时展示本次消耗。
- 自动用药关闭时，探险只继承当前血气和精神，不会自动吃药。

### 出怪频率

- 当前落地为每 90 秒结算 1 次遇怪。
- 30 分钟最多预计算 20 场战斗。
- 战斗中死亡时，后续事件不再继续预计算。
- 背包格子或负重装不下下一件物品库掉落时，后续事件不再继续预计算。
- 纳戒掉落和武器掉落不占背包负重。

### 背包满处理

- 背包格子满或负重满时，本次探险预计算提前停止。
- 堆叠上限不足时，也视为背包装不下。
- 已经预计算成功的经验、物品库掉落、装备库掉落、武器掉落会保留到领取阶段。
- 导致背包满的那件物品不会进入掉落结果。
- 仍然必须等 30 分钟冷却到点后，才能发送 `结束探险` 结算。

### 死亡和损失

- 血气归零视为本次探险死亡或重伤。
- 死亡不删除角色。
- 死亡不掉固定装备。
- 死亡不掉已装备武器。
- 死亡不扣源库余额。
- 当前落地规则下，死亡只会提前停止本次预计算，不额外扣背包物品。

### 掉落结构

```text
物品库掉落：当前地点特产、材料、怪物战利品
装备库掉落：恢复类、宝石、技能书
核心掉落：武器、怪物战利品
稀有掉落：特殊词条武器、稀有宝石、稀有技能书
```

- 物品库、装备库和怪物战利品按战斗事件分别判定。
- 武器按整轮探险判定一次，基础概率 35%，探险加成最高提高到 55%。
- 一次探险最多获得 1 把武器，避免多场战斗把武器掉率放大。

## 十一、战斗系统

### 攻击原则

```text
最终攻击表现 = 人物低基础攻击 + 武器攻击 + 武器自动技能 + 武器附魔效果
```

- 人物等级不提供大量攻击。
- 固定装备不提供攻击。
- 固定装备宝石不提供攻击。
- 技能只跟武器和附魔有关。

### 武器技能

武器 0 级自带 1 个技能。

技能按攻击次数自动释放：

```text
普通攻击计数 -> 达到技能间隔 -> 精神足够则释放 -> 精神不足则跳过
```

### 防御

```text
数值减伤 = min(防御 × 20%, 原始伤害 × 50%)
中间伤害 = 原始伤害 - 数值减伤
免伤率 = 防御 / (防御 + 300 + 30 × sqrt(防御))
免伤率最高 60%
最终伤害 = max(1, 中间伤害 × (1 - 免伤率))
```

穿透：

```text
有效防御 = 防御 × (1 - 穿透率)
```

穿透只来自武器、武器词条和武器附魔。

## 十二、玩家对战系统

### 切磋

切磋是无损 PVP，用来测试战力。

规则：

- 双方都必须已经创建用户。
- 双方都必须处于空闲状态。
- 发起后等待对方接受，10 分钟未接受自动失效。
- 战斗使用双方当前等级、固定装备、当前武器和附魔。
- 战斗结算使用临时血气和精神，不写回玩家当前状态。
- 不消耗背包和纳戒物品。
- 不改变真实血气、精神、源石和物品。
- 不产生经验和掉落。

### 赌约决斗

赌约决斗是带源石押注的 PVP。

规则：

- 双方都必须已经创建用户。
- 双方都必须处于空闲状态。
- 发起人先冻结押注源石。
- 对方接受时冻结同额源石，并立即结算战斗。
- 战斗结算使用临时血气和精神，不写回玩家当前状态。
- 胜者获得奖池扣除手续费后的源石。
- 失败者不掉物品，不掉经验。
- 拒绝、超时或任一方余额不足时，已冻结源石原路退回。
- 同一玩家同一时间只能存在一个未完成赌约。
- 赌约对象可以用 id 或 `CQ/at`，不能用展示名称。

赌约手续费：

```text
奖池 = 押注 × 2
手续费 = floor(奖池 × 3%)
胜者获得 = 奖池 - 手续费
```

### 固定装备升级

```text
固定装备升级源石 = floor(800 × 下一等级 ^ 1.55 × 部位系数)
```

| 部位 | 系数 |
|---|---:|
| 头部 | 1.0 |
| 左手 | 0.9 |
| 右手 | 0.9 |
| 左脚 | 0.8 |
| 右脚 | 0.8 |
| 饰品 | 1.2 |
| 护甲 | 1.5 |

固定装备最高 100 级。

规则：

- 每个固定装备位默认 3 个孔位，最高 9 个孔位。
- 4 到 9 号孔需要使用开孔器逐个开启。
- 开孔器属于消耗品，只通过活动 Boss 掉落，不进入普通探险掉落池。
- 已开启孔位可以镶嵌宝石，未开启孔位不能镶嵌。
- 已有宝石的孔位不能重复镶嵌，需要先拆卸。
- 拆卸后宝石回到纳戒，不会损坏。

固定装备等级会直接回流玩家面板：

```text
固定装备血气加成 = 装备等级 × 8 × 部位系数
固定装备精神加成 = 装备等级 × 3 × 部位系数
固定装备防御加成 = 装备等级 × 2 × 部位系数
```

宝石镶嵌和体质特性都会回流玩家属性：

- `max_hp_bonus` 增加血气上限。
- `mp_bonus` 和 `max_mp_bonus` 增加精神上限。
- `defense_bonus` 增加防御。
- `dodge_bonus` 和 `crit_resist_bonus` 参与战斗承伤。
- `recover_bonus` 提高休息时的精神恢复量。
- `explore_bonus` 小幅提高探险地点特产、纳戒物品和整轮武器掉落机会。
- `trade_bonus` 小幅降低商场买卖手续费。

武器分两层：

- `weapon_defs` 是武器模板，定义名称、基础攻击、可掉落地点、技能类型。
- `player_weapons` 是玩家真正拥有的武器实例，记录随机等级上限、当前等级和附魔。

### 附魔规则

- 武器 0 级自带 1 个技能。
- 附魔栏和武器品级独立，只看 `max_level` 和当前 `level`。
- 掉落时 `max_level` 越高，后续可培养空间越大。
- 附魔使用技能书。
- 附魔后永久占用附魔栏。
- 附魔不可撤销、不可拆卸、不可取消。

附魔栏解锁曲线：

| 条件 | 解锁结果 |
|---|---|
| `max_level >= 20` 且 `level >= 20` | 第 1 个附魔栏 |
| `max_level >= 40` 且 `level >= 40` | 第 2 个附魔栏 |
| `max_level >= 60` 且 `level >= 60` | 第 3 个附魔栏 |
| `max_level >= 80` 且 `level >= 80` | 第 4 个附魔栏 |
| `max_level >= 100` 且 `level >= 100` | 第 5 个附魔栏 |

这样低上限武器不会被迫长期投入，高上限武器每 20 级稳定多 1 个附魔栏。

### 武器升级

```text
武器升级源石 = floor(1500 × 下一等级 ^ 1.6 × 品质系数)
```

| 品质 | 系数 |
|---|---:|
| 凡品 | 1.0 |
| 良品 | 1.4 |
| 珍品 | 2.0 |
| 稀品 | 3.0 |

武器实例不能超过自身 `max_level`，全服最高上限为 100 级。

规则：

- 升级时会按当前等级重新计算已解锁附魔栏。
- 备用武器可以在铸剑阁用 `回收武器 武器ID` 高价回收。
- 已装备武器和最后一把武器不能回收。

### 掉落等级上限

武器掉落时立即生成 `max_level`，之后不再变化。

| 探险地点等级段 | 随机等级上限 |
|---|---|
| 1 到 20 | 20 到 45 |
| 21 到 40 | 35 到 65 |
| 41 到 60 | 50 到 80 |
| 61 到 80 | 65 到 95 |
| 81 到 100 | 80 到 100 |

补充规则：

- 品质影响基础攻击、词条强度和掉落稀有度。
- `max_level` 不读取品质，凡品也可能有高上限，稀品也可能上限一般。
- 高品质低上限适合短期过渡，低品质高上限适合长期培养。

## 十五、宝石

宝石只用于固定装备，不用于武器。

```text
宝石升级源石 = 5000 × 下一等级 ^ 2
```

- 宝石最高 10 级。
- 宝石不提供攻击力。
- 镶嵌命令镶嵌的是宝石。
- 每个固定装备位默认 3 孔，最高 9 孔。
- 孔位命令：`孔位 装备位`。
- 开孔命令：`开孔 装备位`，消耗 1 个开孔器。
- 镶嵌命令：`镶嵌 装备位 孔位号 宝石名称`。
- 同名不同等级宝石可以写：`镶嵌 装备位 孔位号 宝石名称 2级`。
- 纳戒里只有一个等级时可省略等级；同名存在多个等级时必须写等级。
- 拆卸命令：`拆卸 装备位 孔位号`。
- 宝石升级命令：`宝石升级 装备位 孔位号`。
- `我的宝石` 查看纳戒中尚未镶嵌、可用于镶嵌的宝石库存。
- 已镶嵌宝石以装备位和孔位号作为唯一位置，避免同名宝石无法区分。
- 当前代码拆卸宝石免费，且不会损坏宝石。

## 十九、实现约束

这些约束不属于玩法展示，但写代码时必须统一。

### 公共函数

根目录公共模块建议至少提供：

```text
business_day(now)        # 按每日 04:00 计算业务日
parse_player_ref(text)   # 把纯 id 或 CQ/at 转成 client_id
validate_name(name)      # 校验创建用户和改名的展示名称
require_player(client_id) # 确认玩家存在，不存在则提示创建用户
check_request_id(...)    # 防止 request_id 重复执行
weapon_core             # 武器读写、初始武器和掉落武器
combat_core             # 打怪和玩家对战结算
item_effects            # 可使用物品效果
```

### 数据写入

- 源石、背包、纳戒、武器、市场、赌约都必须在数据库事务里完成。
- 扣钱和发货必须同一事务，不能先扣钱后失败。
- 二手市场购买、商场出售、赌约接受都要重新读取最新数据后再结算。
- 同一玩家的状态修改建议加轻量锁，避免连续命令同时改背包或源石。
- 带 `request_id` 的 WS 请求，执行前先查 `request_idempotency`。
- 同一 `client_id + request_id` 已完成时，不再重复执行，只返回上次结果。

### 日志和记录

- 经济变动写入 `game_logs`，便于后期排查。
- 战斗详情只在 `combat_logs` 记录摘要，不存过长文本。
- 探险结算以 `exploration_records` 为准，领取后不可重复领取。
- 赌约源石冻结和退回都记录到 `duel_records` 或 `game_logs`。

### 当前可优化点

- `request_id` 幂等表正式启用，防止同一条 WS 请求被客户端重试后重复执行。
- 商场记录库存热度，买得越多短期越贵，卖得越多短期越便宜。
- 探险地点显示推荐等级和怪物等级段，但不限制进入。
- 武器技能只按固定倍率、精神消耗和触发间隔结算。
