# 玩家组件说明

玩家组件负责创建角色、展示资料、改名、签到、新手礼包、自动用药开关和休息恢复。

## 命令

```text
帮助 / 修仙帮助
指南
创建用户 名称
改名 新名称
修仙信息
状态
修仙日记
自动用药
自动用药 开启 / 自动用药 关闭
战斗日志
战斗日志 开启 / 战斗日志 关闭
签到
新手礼包
休息
结束休息 / 休息结束
```

## 规则

- 玩家必须先创建用户，名称只用于展示和指令查找，不能重复。
- 名称长度为 2 到 12 个字符，不能包含空白字符。
- 改名有 24 小时冷却。
- `帮助` 和 `修仙帮助` 优先返回 `玩家/help.png` 的 base64 图片；图片不存在时返回文字指引。
- `指南` 返回关键二级组件按钮入口，只放固定命令，不放需要用户补参数的命令。
- `修仙信息` 展示称号、等级、经验、血气、精神、体质、攻击、防御、速度、技能节奏、武器定位、武器技能、装备、今日加成、源石、状态、自动用药、战斗日志模式和地点；有仇恨关系时额外展示死敌。
- `状态` 只展示关键数据：等级经验、血气精神、当前状态、地点、源石、攻击、防御、速度、技能节奏、当前武器、自动用药、战斗日志、今日加成合计；有仇恨关系时额外展示死敌。
- 面板字段来源：等级、经验、源石、状态、自动用药、战斗日志模式、地点来自玩家档案。
- 称号按当前数据自动分配并自动佩戴；没有称号时显示 `无`。
- 血气上限 = `100 + 等级 × 12 + 体质值 × 8 + 装备/宝石/体质/气运/天气/灵潮加成`。
- 精神上限 = `60 + 等级 × 6 + 装备/宝石/体质/气运/天气/灵潮加成`。
- 防御 = `等级 + 体质值 × 2 + 装备/宝石/体质/气运/天气/灵潮加成`。
- 面板攻击力 = 玩家基础攻击 + 当前武器攻击；武器技能和附魔效果在战斗结算时另外生效。
- 面板速度来自等级、武器类型、武器攻击负重、命中/闪避类效果；轻武器更快，重武器更慢。
- 技能节奏来自武器自带技能蓄势基准、武器类型和当前速度，表示实战中大约多久能蓄满一次技能。
- 体质资料来自体质库，当前武器来自玩家武器库和武器模板。
- 武器技能会显示当前武器的自带技能和已附魔技能；铭刻后的技能名会优先展示，并保留原名。
- 装备摘要来自装备表，会显示每个装备的展示名、等级和已开孔数。
- 今日加成会展示签到气运、全服天气、今日灵潮和合计效果；未签到时气运显示为未签到。
- `自动用药` 不带参数时查看状态；支持开启、打开、启用、开、on、ON、1，也支持关闭、关掉、停用、关、off、OFF、0。
- 自动用药只在探险预计算时生效，会消耗纳戒里的恢复类物品。
- `战斗日志` 不带参数时查看当前模式；默认是简要。
- `战斗日志 开启` 切到详细逐次出手日志，`战斗日志 关闭` 切回简要摘要；也支持打开、启用、开、on、ON、1 和关闭、关掉、停用、关、off、OFF、0。
- 战斗日志模式影响探险、虫洞、首领、切磋、决斗和抢劫的结算展示。
- 有仇恨关系时，`修仙信息` 和 `状态` 才展示死敌；死敌表示当前对你仇恨最高的玩家。报复指数 = 仇恨值 x 20，最高 100。被抢后仇恨会上升，反抢复仇成功后会清空对应仇恨。
- `修仙日记` 按玩家当前数据补齐并刷新里程碑，不记录每一步流水。
- 日记会覆盖创建、等级、资产、地点状态、签到、礼包、改名、探险、跑商、二手市场、武器、回收、虫洞、首领、对战、铭刻和物品使用等长期记录。
- 签到每天 04:00 后刷新一次，并生成或展示今日气运。
- 今日气运只从签到入口查看，当天生效，数值大约在 0 到 10% 之间。
- 全服天气和灵潮按业务日稳定生成，在修仙早报和修仙信息中展示，并会实际影响对应玩法。
- 新手礼包只能领取一次，奖励源石 10000、血契丹 2、阴冥草 2。
- 休息要求玩家处于空闲状态，持续 1 分钟。
- `结束休息` 和 `休息结束` 都是精确命令，休息时间到后恢复血气并恢复部分精神。

## 扩展

- 创建、改名和玩家名解析放在 `common.py` 的 `CoreService` 中，其他组件统一复用。
- 调整新手礼包源石：改 `NEWBIE_GIFT_STONES`。
- 调整休息时间：改 `REST_MINUTES`。
- 调整签到奖励：改 `rules.py` 的 `sign_reward()`。
