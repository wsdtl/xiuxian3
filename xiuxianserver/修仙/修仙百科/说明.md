# 修仙百科组件说明

修仙百科负责把修仙设定、组件说明和 `xiuxian.db` 中的结构化资料整理成玩家助手。它不是只解释词条，还要把玩家带到下一步可执行操作。

## 命令

```text
修仙百科 问题
```

## 规则

- `修仙百科 问题` 会从启动时缓存的稳定 Markdown 文档和数据库结构化资料中检索答案。
- Markdown 是一等知识源：`完整设定.md`、`architecture.md`、`富文本规则.md`、各组件 `说明.md`、世界物资设定、数值统筹规划和平衡表会按章节切片入库；封版记录、实施方案、绘图参考和一次性整理文档不进入百科。
- 百科会保留文档来源、组件归属和章节路径；综合问题会跨多个章节整理结论、规则、流程、入口、数值和来源，并在末尾列出参考文档。
- 数据库来源为当前修仙库 `修仙/xiuxian.db`，只读不写，不新增表。
- 缓存内容包含物品、恢复类消耗品、宝石、技能书、武器、武器技能、附魔效果、跑商地点、探险地点、怪物掉落、特殊收购、回收地点、首领奖励概率和当前世界皮肤名录。
- 当前世界皮肤名录来自修仙根目录 `world_skin.py`，包含当前展示名、稳定 ID、分类和简要说明；百科回答优先使用当前展示名，命令建议仍使用固定命令词。
- 命令触发时优先读内存缓存，不递归扫描文件，也不反复全表读取数据库；缓存为空时才重建知识快照。
- 修改 Markdown 或普通数据库定义后，需要重启项目，百科缓存才会刷新；世界皮肤切换会通过运行态缓存总线清空百科快照，下一次查询自动按当前皮肤重建。
- 当前为规则合成式百科，不调用外部大模型；命中资料后会按问题意图整理成结论、用途、来源、流程或玩法建议，不直接展示搜索结果列表。
- 具体物品问“怎么用 / 有什么用”时，百科必须优先回答真实入口、当前库存、可用目标和下一步命令；不能只说“这是消耗品”或“不要乱用”。
- 开孔器、洗髓液、淬锋丹、宝石、技能书、恢复类物品会读取玩家当前库存、装备孔位或武器状态，给出是否能立即操作、应发送什么命令、缺什么去哪里补。
- 具体实体问题优先使用数据库结构化资料，例如某个宝石、技能书、武器、地点或怪物；宽泛机制问题优先综合 Markdown，例如宗门大会奖励机制、按钮规则、世界物资闭环。
- 如果 Markdown 默认示例名和当前皮肤展示名不一致，具体实体问答以当前世界皮肤名录和数据库结构化资料为准。
- 武器和技能书问题会按 `日常探险 / 首领虫洞 / 决斗抢劫 / 越级生存 / 开荒` 给出场景评分，并补充最优方案、替代方案和不建议方案。
- 带 `我、我的、当前、现在` 的推荐类问题会读取当前玩家状态和已装备武器；如果没有角色或没有装备武器，会先给出下一步提示。
- 玩家自己的武器铭刻名、自带技能铭刻名和已附魔技能书铭刻名可以参与百科识别；显示规则为 `铭刻名（原名）`。
- 回复末尾按钮会根据问题主题生成，不固定使用同一组按钮。

## 组件关联

- 百科启动时读取稳定 Markdown 和数据库结构化资料；组件说明写清联动关系后，会直接影响百科综合回答。
- 百科与修仙帮助共用当前世界名录，保证帮助搜索和百科问答看到同一套皮肤展示名；世界皮肤切换后百科不会继续使用旧皮肤快照。
- 具体物品、武器、技能书、地点、怪物、首领和奖励优先来自数据库；宽泛机制问题优先综合设定文档。
- 推荐类和具体物品用法问题会读取玩家当前状态、当前武器、装备孔位、库存、附魔和铭刻名，但只做解释与建议，不修改任何业务数据。
- 新增组件、物品、数值、命令或联动后，应先更新对应 Markdown 和数据库种子，再重启让百科缓存刷新。

## 维护边界

- 新增结构化知识来源：改 `service.py` 的 `_load_database_entries()`。
- 新增设定知识：优先写入对应 `.md`，重启后自动被百科按章节收录。
- 调整检索规则：改 `service.py` 的 `_search()`、`_query_terms()` 和 `_should_use_document_synthesis()`。
- 调整综合回答：改 `service.py` 的 `_document_synthesis_lines()`、`_rank_document_sentences()` 和 `_document_topic()`。
