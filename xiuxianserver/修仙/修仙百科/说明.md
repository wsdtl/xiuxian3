# 修仙百科组件说明

修仙百科负责把修仙设定、组件说明和 `xiuxian.db` 中的结构化资料整理成可查询知识库。

## 命令

```text
修仙百科 问题
```

## 规则

- `修仙百科 问题` 会从启动时缓存的 Markdown 文档和数据库结构化资料中检索答案。
- 数据库来源为当前修仙库 `修仙/xiuxian.db`，只读不写，不新增表。
- 缓存内容包含物品、恢复类消耗品、宝石、技能书、武器、武器技能、附魔效果、跑商地点、探险地点、怪物掉落、特殊收购、回收地点和首领奖励概率。
- 命令触发时只读内存缓存，不递归扫描文件，也不反复全表读取数据库。
- 修改 Markdown 或数据库配置后，需要重启项目，百科缓存才会刷新。
- 当前为规则合成式百科，不调用外部大模型；命中资料后会按问题意图整理成结论、用途、来源或玩法建议，不直接展示搜索结果列表。
- 武器和技能书问题会按 `日常探险 / 首领虫洞 / 决斗抢劫 / 越级生存 / 开荒` 给出场景评分，并补充最优方案、替代方案和不建议方案。
- 带 `我、我的、当前、现在` 的推荐类问题会读取当前玩家状态和已装备武器；如果没有角色或没有装备武器，会先给出下一步提示。
- 玩家自己的武器铭刻名、自带技能铭刻名和已附魔技能书铭刻名可以参与百科识别；显示规则为 `铭刻名（原名）`。
- 回复末尾按钮会根据问题主题生成，不再固定使用同一组按钮。

## 扩展

- 新增知识来源：改 `service.py` 的 `_load_database_entries()`。
- 调整检索规则：改 `service.py` 的 `_search()` 和 `_query_terms()`。
- 调整回答条数：改 `MAX_RESULTS`。
