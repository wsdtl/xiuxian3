# 修仙物品查看组件说明

修仙物品组件提供统一的物品定义查询入口。它查的是配置资料，不是玩家库存。

物品设定先按 `世界物资设定.md` 收口：世界物资和角色成长道具分家，六大类世界物资只服务经济、地点、补给、建设、回收和掉落流动；武器、技能书、宝石、恢复药、开孔器、洗髓液、淬锋丹、铭刻之羽和源石不进入六大类。

## 命令

```text
查看修仙物品 物品名
修仙物品查看 物品名
查看 物品名
```

## 可查看内容

- 世界物资：纯经济、药路、民生、建设、古物、战利品六大类 `item_defs` 物品。
- 角色成长道具：恢复类、宝石、技能书、消耗品和专属道具等 `ring_item_defs` 物品。
- 武器模板：武器类型、掉落地点、模板基础攻击和自带技能。
- 武器自带技能：威力、精神消耗、蓄势基准和说明。
- 技能书附魔效果：具体战斗效果和精神消耗变化。
- 体质资料：阶位、路线、体质值、稀有等级、效果和说明。

## 规则

- 玩家必须已创建用户才能查询。
- 这里只看定义，不看玩家身上有没有该物品。
- 武器实例有等级、攻击、附魔和自定义名，必须使用 `查看武器` 或 `查看武器 武器ID`。
- 铭刻后的武器名和附魔名属于实例显示名，不会改变 `查看修仙物品` 查到的基础定义。
- 找不到物品时，会提示先从 `背包`、`纳戒`、`武器` 复制准确名称。
- 世界物资详细分类和闭环以 `世界物资设定.md` 为准；业务代码落地时不要绕开这份设定。
- 查询世界物资时，需要展示它的六大类、子类、主要来源和当前去路：纯经济走商场买卖，药路/民生/建设/古物走 `出售` / `自动出售` 转入城池，战利品走同一入口转入特殊收购。

## 组件关联

- 本组件读取 `item_defs`、`ring_item_defs`、武器、技能、附魔和体质定义，是定义查询入口，不读玩家库存和实例状态。
- 背包、纳戒、武器、装备、铭刻、贸易服务、探险、首领和虫洞新增物品或效果时，都要保证这里能查到同一套口径。
- 世界物资分类以 `世界物资设定.md` 和数据库效果字段为准，商场出售、探险掉落、城池状态、藏宝图和战备虫洞都依赖这个分类。
- 铭刻名只属于玩家实例；基础定义仍显示原始物品、武器、技能和附魔信息。

## 扩展

- 新增可查询定义时，优先在本服务补一个按名称读取和格式化方法。
- 展示效果统一使用 `format_effect()`，避免不同组件对同一效果写出不同文本。
- 更新世界物资分类时，先改 `世界物资设定.md`，确认纯经济 33 个地区特产不被其他大类挤占，再改初始化数据和查询展示。
