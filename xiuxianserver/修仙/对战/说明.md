# 对战组件说明

对战组件负责玩家之间的切磋、押注决斗、请求接受拒绝和最近决斗记录。

## 命令

```text
切磋 对方名称 / 切磋 [CQ:at,qq=对方ID]
接受切磋 发起人名称 / 接受切磋 [CQ:at,qq=发起人ID]
拒绝切磋 发起人名称 / 拒绝切磋 [CQ:at,qq=发起人ID]
决斗 货币数量 对方名称 / 决斗 [CQ:at,qq=对方ID] 货币数量
接受决斗 发起人名称 / 接受决斗 [CQ:at,qq=发起人ID]
拒绝决斗 发起人名称 / 拒绝决斗 [CQ:at,qq=发起人ID]
抢劫 对方名称 / 抢劫 [CQ:at,qq=对方ID]
决斗记录
```

## 规则

- 涉及对方或发起人的命令优先写玩家名称，也可以直接使用 CQ/at；WS 会把 CQ/at 转成原始入口 id，业务层再通过用户组解析到主角色 id。
- `切磋` 是无押注对战。
- `决斗` 必须带正整数货币数量和对方。
- 决斗参数支持金额和对方互换顺序，但推荐写法是 `决斗 货币数量 对方名称`。
- 发起请求和接受请求时，双方都必须处于空闲状态，不能正在探险、休息或参与其他占用本体的战斗。
- 每个玩家同一时间只能有一个未处理对战请求。
- 对战请求 10 分钟后超时。
- 如果接受时任意一方已经不空闲，本次不会结算，未过期请求仍保留；双方恢复空闲后可继续接受，或手动拒绝。
- 决斗发起时先冻结发起人的押注货币；接受时再扣接受人的押注。
- 如果接受人货币不足，决斗取消并退回发起人冻结货币。
- 胜者获得双方押注扣除 3% 手续费后的奖池。
- 接受成功后立即结算，默认返回短结算和战斗日志链接；消息只展示胜负、双方剩余血气精神、武器经验和押注结算，行动次数与技能统计进入网页日志。
- 发送 `战斗日志 开启` 后，消息仍保持短摘要，但战斗日志网页默认展开逐次出手；发送 `战斗日志 关闭` 后网页默认折叠逐次出手。
- 战败后精神归零。
- `抢劫` 是即时对战，不需要对方同意，只能抢正在探险中的玩家。
- 抢劫方使用当前真实数据，被抢方使用探险开始时写入的战斗快照，被抢方状态不变。
- 抢劫成功才会从对方本轮探险预计算战利品里拿走物品；没有战利品则只记录抢劫结果。
- 同一轮探险最多被抢劫 2 次，同一个抢劫方不能重复抢同一轮探险。
- 抢劫成功后，被抢方对抢劫方仇恨 +1，报复指数 +20；仇恨表示“我想抢回来”的强度，会在抢劫方的 `修仙信息` 里以死敌展示。
- 如果抢劫方原本对目标有仇恨，本次胜利会触发复仇加成，最多额外拿 5 件战利品，并清空这条仇恨。
- 对战网页日志读取 `duel_records.result` 和 `robbery_records.result`；切磋、决斗、抢劫流水先汇总到长期统计，再按 7 天清理。
- `决斗记录` 只展示最近 10 条，避免消息窗口过长；清理每天最多执行一次。

## 组件关联

- 对战使用公共战斗核心、玩家当前武器/装备/体质/宝石加成和战斗日志网页。
- 切磋和决斗要求双方空闲；抢劫只能针对探险中的玩家，并读取目标探险开始快照。
- 抢劫成功会从目标探险预计算结果中转移战利品到抢劫方库存，并写入仇恨、抢劫记录和可能的宗门大会影响力。
- 决斗押注使用随身货币，不直接从银行扣款；胜者获得扣手续费后的奖池。
- 对战结果会影响武器经验、称号、修仙日记、人物志、修仙界历史和长期统计；普通切磋/决斗/抢劫不刷新休息恢复窗口。

## 维护边界

- 对战核心结算使用根目录 `combat_core.py`，不是二级包互相导入。
- 修改请求有效期：改 SQL 插入时的 `+10 minutes`。
- 调整决斗手续费：改 `DuelService._accept()` 里的费率。
- 调整战斗日志：短摘要改 `DuelService._duel_log_block()` 和 `combat_log_text.py`，网页逐回合展示改 `战斗日志/site.py`。
