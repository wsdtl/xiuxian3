# 修仙完整设定

本文按当前代码整理，冲突时以代码为准。修仙模块以 WS 命令为主要玩法入口，数据库、帮助图、Markdown 文档站和数据备份都属于玩法边界的一部分。

## 一、模块边界

组件边界：

- 修仙帮助：帮助网页、修仙帮助图、指南按钮、当前世界资料索引和 Markdown 文档站。
- 修仙百科：`修仙百科 问题` 问答入口，启动时缓存 Markdown 设定、当前世界皮肤名录和 `xiuxian.db` 中的物品、武器、技能、地点、怪物、首领奖励等结构化资料。
- 后台接口：承接 web 后台 API，现阶段不注册玩家命令。
- 用户组：承接多入口身份映射、用户组后台登录页面、登录码确认和绑定码流程，不做已有角色资产合并。
- 玩家：创建用户、改名、修仙信息、状态、修仙日记、签到、新手礼包、自动用药、休息，并在有仇恨关系时展示死敌和报复指数，同时展示今日气运等可追踪加成。
- 背包：占负重物品库存，恢复类物品使用入口。
- 纳戒：不占负重物品库存，承接体质重塑、淬锋丹、开孔器等特殊纳戒物品消耗。
- 保险箱：冻结保存背包、纳戒、宝石和备用武器，避免出售、回收或上架误操作。
- 修仙物品：统一查看修仙物品定义。
- 缘契：消耗缘契凭证生成一次性开启码，进入随机或指定角色剧情体验页。
- 洞天福地：异世小游戏入口，静态网页放在 `static/dongtian`，服务端签发一次性兑换码并兑换少量修仙资源。
- 银行：货币存储、结息、升级、存入、取出。
- 二手市场：玩家间一包商品交易，支持背包物品、纳戒物品、宝石、武器。
- 贸易服务：跑商交易、统一出售、自动出售、导航、藏宝图和每日跑商奖励。
- 宗门：建立宗门、山门查看、加入/退出宗门、宗门大会影响力、个人贡献和淬锋丹奖励。
- 探险：普通探险 30 分钟预计算；太虚秘境按实际战斗场数计时。自动战斗、自动用药、掉落、默认简要结算消息，并提供战斗日志网页链接。
- 武器：武器列表、详情、武器传奇、切换、升级、附魔。
- 装备：装备、镶嵌、拆卸、宝石升级。
- 铭刻：装备、武器、自带技能、武器附魔改显示名。
- 对战：切磋、押注决斗、抢劫、仇恨、接受、拒绝、战斗记录。
- 异界虫洞：跑商触发的动态世界 Boss。
- 首领：每天都会出现的岁时情劫；二十四节气和传统节日优先，普通日从每日旧愿首领池轮换。
- 修仙界历史：风云榜、修仙早报、修仙界历史、史册分卷、公开人物志，并公开展示人物、商会、城池和首领动态。
- 世界皮肤：提供世界皮肤包查看、校验和主人级热切换；只替换展示名，不改变命令、稳定 ID、数值公式和玩家资产。
- 数据库备份：服务关闭时自动备份修仙 sqlite 数据库。

工程约束：

- 命令入口以当前代码显式注册为准；代码中保留的别名也属于正式入口，不支持未注册命令。
- 中文只用于业务包名和玩家可见文本。
- 二级包之间不互相导入，共享能力必须下沉到修仙根目录公共模块。
- HTTP 路由包含 `修仙帮助` 文档站、`用户组` 绑定后台和 `后台接口`；玩家玩法主要通过 WS 命令触发。
- 数据库结构以 `SCHEMA_VERSION = 2026062805` 为准，不维护历史结构兼容包袱；地点、特殊收购建筑、回收建筑、纯经济特产、纳戒物品、品质、货币和等级显示已拆分稳定 ID 与展示名，世界皮肤包只替换展示名；用户组表只承接入口身份映射，不做已有角色资产合并。
- 世界皮肤切换属于主人级操作，权限名单来自 `.env` 的 `MASTER_NAME`，命令入口通过 `Depends(current_is_master)` 按玩家显示名注入权限结果，普通玩家不能切换皮肤包；切换命令会先自动校验，失败不写库；开孔器、洗髓液、淬锋丹、流光签作为特殊纳戒道具进世界皮肤包，业务仍只认稳定 `ring_item_id`；岁时情劫和铭刻之羽不进入世界皮肤包。

组件联动总则：

- 回复层是所有 WS 组件的统一出口：text/markdown 回复都由 `send_reply` 补玩家头、系统消息队列、个人通知栏和按钮；手写 `<命令>` 按钮永远优先。
- `修仙帮助` 和 `修仙百科` 会递归读取稳定 Markdown；组件说明、完整设定、架构、富文本规则、世界物资设定、数值统筹规划和平衡表都会变成玩家可见知识。两者还会读取当前世界皮肤名录，具体实体展示优先使用当前皮肤名，命令词保持固定。
- 背包、纳戒、保险箱、武器库、装备和原石各有边界：背包承接占负重世界物资，纳戒承接不占负重成长道具，保险箱冻结资产，武器是独立实例，装备是固定长期成长，银行只是货币存储。
- 贸易服务是交易、出售分流、导航和藏宝图入口；探险负责位置展示、探险列表、战斗预计算、掉落、休息窗口和战斗日志链接；修仙帮助负责帮助站和交互地图网页。
- 世界物资统一由 `world_materials.py`、商场、探险、虫洞和修仙界历史共同消费：纯经济走跑商，药路/民生/建设/古物流入城池，战利品流入特殊收购和战备虫洞。
- 宗门不承接普通世界物资回收，但会读取宗门成员行为、宗门大会影响力、城池范围增益和世界物资沉淀；淬锋丹奖励进入纳戒，再由 `武器升限` 作用到武器。
- 探险、对战、首领和虫洞都使用公共战斗核心、公共加成汇总和战斗日志网页；武器经验、人物经验、掉落、宗门影响、称号和历史沉淀分别由对应组件消费。
- 通知、称号、日记、人物志、早报和历史是被动反映层，只读取已有业务事实，不新增复杂主动命令，也不替代各玩法的领取或结算入口。

## 二、通讯规则

WS 连接：

```text
/ws/bot/{client_id}
```

标准消息：

```json
{
  "code": 202,
  "type": "text",
  "message": "修仙信息",
  "request_id": "本次请求唯一 id"
}
```

返回格式：

```json
{
  "code": 202,
  "type": "markdown",
  "message": {
    "content": "【玩家名·称号 LV等级】\n🔴 系统：全服事件队列\n🔴 通知：个人待办队列\n回复文本",
    "keyboard": {}
  },
  "request_id": "原 request_id"
}
```

通讯约定：

- 正常业务回复使用 `code=202`。
- 协议错误、通讯错误、未命中触发器使用 `code=404`。
- `type` 由业务自己决定，常用 `text`、`image` 和 `markdown`。
- 图片回复的 `message` 是纯 base64 字符串，不带 `data:image/...` 头。
- `markdown` 回复的 `message` 是对象，里面包含 `content` 和可选 `keyboard`。
- `request_id` 是客户端等待回复的凭证，请求和回复必须对应。
- 文本和 markdown 回复由 `send_reply` 自动加回复头，顺序固定为玩家身份、可选系统消息队列、可选个人通知栏；图片回复不额外加标题。
- 系统消息队列使用 `🔴 系统：`，只放全服/世界层面的短事实，例如开启中的战备虫洞、普通虫洞、岁时情劫、宗门大会领取日/收官日、即将结束的藏宝图拍卖和散落藏宝图；队列项使用无框命令链接，点击后进入对应玩法或状态页。系统消息展示中短 `label`，保留事件类型和地点或核心识别物，普通/战备虫洞优先展示地点。
- 系统消息是世界事实队列，不是个人已读/已处理队列；它不会因为某个玩家点开一次就隐藏，只会随事件结束、过期、优先级变化或世界状态变化而自然消失。
- 系统消息按优先级排队，默认只展示队首 3 条；前面的事件过期或消失后，后面的事件自然补上，不单独建公告命令。
- 个人通知栏使用 `🔴 通知：`，只放和当前玩家有关、需要处理、会错过或刚发生且应被看见的事项，例如休息可结束、探险可结束、首领/虫洞/宗门大会/跑商奖励待领、对战请求、藏宝图待领、自己竞拍的藏宝图将结、二手市场成交到账，以及低优先级的新手礼包待领、银行计息达到 24 小时上限、今日签到待领；通知项使用无框命令链接，点击后自动发送对应命令。个人通知展示中短 `label`，保留玩法名和动作，去掉 `待领`、`可领`、`待处理` 这类重复尾巴。
- 系统消息和个人通知都不能使用尖括号按钮标记，避免污染业务手写按钮和正文预测按钮；容易误操作的事项只链接到状态或记录面板，不自动确认不可逆动作。
- 修仙业务函数可以返回普通文本；最终发送时，text/markdown 回复都会被回复层包装成 `markdown`，并追加玩家头和必要按钮。
- 触发失败、条件不满足、需要提示下一步时，`T.hint()` 会调用 `T.tip()` 生成斜体提示，并把建议原文放到正文末尾。
- 建议文案按用户能读懂的自然句写，例如 `发送：探险列表 查看地点，再发送：导航 地点名`。
- 回复层不从带占位参数的建议文案里猜按钮；没有 `<命令>` 标记时，也会因为预测按钮、当前组件按钮或默认按钮统一作为 `markdown` 发送。
- 想生成按钮时由业务手写 `<命令>`，例如 `血气不足，可以先<休息>`。
- 尖括号里的内容会原样作为按钮命令，是否能点击执行由业务自己控制。
- 需要显示文字和实际命令不同时，使用 `<实际命令:显示文字>`，例如 `<商场推荐:去商场>`。
- 默认按钮来自 `reply.py` 的 `DEFAULT_BUTTONS`，当前为 `指南`、`状态`、`修仙信息`。
- 预测按钮来自回复正文关键词，只生成固定命令，例如血气不足给 `休息`，货币不足给 `银行`，背包不足给 `自动出售`，单次最多先取 3 个。
- 当前组件按钮来自 `reply.py` 的 `CONTEXT_BUTTONS_BY_GROUP`，只用于自动补足，不替代业务自己写好的入口。
- 默认按钮只在没有手写、预测和当前组件按钮时出现。
- 最终按钮顺序为业务手写按钮、预测按钮、当前组件按钮、默认按钮，并按实际发送命令去重。
- 业务手写按钮页最多保留 15 个；普通自动补按钮目标为 6 个以内。
- `指南` 只展示方向按钮，具体入口下沉到成长、账户、行囊、奖励、武器、装备、铭刻、探险、战斗、首领、交易、出售、宗门和世界等方向页；探险列表、装备孔位等主动打开的入口页可以展示完整按钮；失败提示、状态回执、结算回执以少而准为准。
- `指南` 和帮助图这类导航页不自动补当前组件按钮和默认按钮。

命令拆分：

- 精确命令取第一个空格前文本为 `cmd`。
- 第一个空格后的原文作为 `message`，不主动改成列表。
- 拆分前，WS 层会把 `[CQ:at,qq=xxx]` 转成 `xxx`，并把多余空白压成单空格。
- 例如 `切磋[CQ:at,qq=xxx]` 会拆成命令 `切磋`，参数 `xxx`。
- 例如 `决斗 1000[CQ:at,qq=xxx]` 会拆成命令 `决斗`，参数 `1000 xxx`。

玩家引用：

- 当前正式指令优先使用玩家名称，因为新接口 id 很长。
- 名称唯一，避免对战、购买、接受请求时混淆。
- 涉及对方的命令也可以直接@对方；WS 层收到平台 @ 码后会转成原始入口 id，业务层再通过用户组映射到主角色 id。

## 三、玩家核心

创建规则：

- 命令：`创建用户 名称`。
- 名称 2 到 12 个字符。
- 名称不能包含空白字符。
- 名称全服唯一。
- 创建后默认 1 级、0 经验、凡体、空闲状态、天枢城位置。
- 创建时自动初始化银行、装备位。

改名规则：

- 命令：`改名 新名称`。
- 名称规则和创建一致。
- 改名冷却 24 小时。
- 名称只用于展示和指令查找，不改变玩家内部标识。

基础常量：

```text
每日刷新：04:00
等级上限：100
默认地点：天枢城
默认背包格子：80
默认负重：500
休息恢复：1 分钟约恢复一半，30 分钟恢复满
玩家基础攻击常量：5
战斗详细日志保留：5 天
直接流水保留：7 天
早报缓存保留：7 天
长期统计：从首次启用清理开始记录，不回填历史流水
```

基础公式：

```text
1-40级升级经验 = floor(120 * level^1.85 + 600 * level)
40级后升级经验 = 基础经验 * (1 + 8*p^1.35 + 12*p^2.4 + 18*p^4.0)，p=(level-40)/59
血气上限 = 100 + 等级 * 12 + 体质值 * 8 + 装备/宝石/体质/气运血气加成
精神上限 = 60 + 等级 * 6 + 装备/宝石/体质/气运精神加成
人物基础攻击 = 5 + floor(等级 / 10)
防御 = 等级 + 体质值 * 2 + 装备/宝石/体质/气运防御加成
签到货币 = min(500 + 等级 * 30, 5000)
```

- 经验型等级到达当前等级上限后，多余经验直接作废，不存为提高上限后的补发经验。
- 玩家经验封顶到当前 `MAX_LEVEL` 所需累计经验；武器经验封顶到该武器当前等级上限所需累计经验；宗门等级和城池建设满级后本级经验归零。

等级体验目标：

- 40 级前较顺滑，方便新玩家建立正反馈。
- 40 级后明显加难。
- 60 级后开始拉开升级差距。
- 80 级后进入长期区，专门压住每日十几次探险的高频玩家。
- 按满胜探险次数估算，1 到 100 级约需要 9000 次完美探险。
- 高等级刷低级怪经验衰减，低等级越级打怪经验提高。

体质系统：

- 默认体质为凡体。
- 洗髓液可重骰体质。
- 高级体质更强调特色，不做万金油。
- 体质效果会进入 `CoreService.equipment_bonuses()`，和装备、宝石、签到气运一起汇总。

休息：

- 玩家必须空闲。
- `休息` 后进入休息中。
- 至少休息 1 分钟才能发送 `结束休息` 或 `休息结束` 结算。
- 按当前恢复窗口的战后/初始状态作为基线结算：前 1 分钟恢复本窗口缺失血气和精神约 50%，后续 29 分钟线性补完，30 分钟后不再增加收益。
- 普通反复短休会沿用同一个窗口累计休息秒数，不会重复吃前 1 分钟收益。
- 结束休息展示的恢复效率按本次实际补回比例计算；恢复增益会参与最终补回，但另行展示为 `恢复增益` 倍率。
- 回春类宝石、体质和气运带来的恢复增益独立结算，按基础恢复量做乘法；展示为 `恢复增益` 倍率，`100%` 表示无修正，低于 `100%` 表示恢复变慢，高于 `100%` 表示恢复变快，不并入恢复效率。
- 固定冷却的战斗结算后会刷新恢复窗口，例如普通探险、虫洞和首领；太虚秘境这类动态领取冷却不刷新，普通切磋、决斗或抢劫也不刷新。

## 四、库存系统

背包：

- 命令：`背包`、`使用 物品名`、`使用 物品名 数量`。
- 保存占负重物品。
- 跑商商品统一为纯经济地点特产；药路、民生、建设、古物和战利品不进入普通跑商特产，怪物战利品和普通掉落进入背包。
- 背包检查格子上限、负重上限、堆叠上限。
- `使用` 只允许直接消耗恢复类物品。
- 不写数量时默认使用 1 个；例如 `使用 福袋 5` 会一次打开 5 个福袋。

纳戒：

- 命令：`纳戒`、`体质重塑`、`武器升限`、`开孔 装备位`。

- 恢复类、福袋、洗髓液、开孔器、流光签、宝石、技能书都进入纳戒。
- 宝石按名称和等级单独存储。
- `体质重塑` 消耗体质重塑道具；默认皮肤中该道具显示为洗髓液。
- `武器升限` 消耗武器升限道具，默认提升已装备武器等级上限；也可发送 `武器升限 武器ID` 提升指定武器，最高 100。

祈愿：

- 命令：`祈愿`、`十连祈愿`、`祈愿奖池`、`我的凭证`、`祈愿记录`。

## 五、物品库

世界物资：

- 世界物资是世界流动、地点消耗、跑商搬运、战斗掉落和回收处理的物品，不直接让玩家变强。
- 六大类为：纯经济、药路、民生、建设、古物、战利品。
- 详细分类、名称、城池承接、药路顺药、民生恩赐、建设等级、古物藏宝图和战备虫洞规则，以 `修仙物品/世界物资设定.md` 为准。
- 纯经济只包含 11 个普通跑商地区各 3 个特产，共 33 个，专门服务地区供需差价。
- 药路包含药材、丹材、炉火，用于解释恢复药补给、顺药概率和商路补给效率；玩家不能通过商场直接买恢复药。
- 民生是城食、盐鲜、水净、衣被、燃安等凡城刚需物资，可回收，不受普通价格曲线调控，并影响当地 `极·技能书` 概率。
- 建设是古界建材、护城灵材、阵基异材和门面装饰，回收后进入当地城池建设经验。
- 古物按神秘物质含量分微蕴、中蕴、厚蕴；神秘物质只有数量多少之分，用于当地藏宝图蓄能。
- 战利品来自妖、魔、鬼、龙、兽、兵戈等战斗掉落，主要走特殊收购和战备虫洞生态，不参与普通跑商。
- 只有天枢城、青岚坊、赤霞港、玄铁岭、万药谷、云梦泽、流沙海市、寒霜关、雷泽城、碧潮岛、星陨墟这 11 个普通探险/跑商重合点承接世界状态。
- 药路、民生、建设、古物通过 `出售` / `自动出售` 流入 11 个承接城池，纯经济走商场买卖，战利品通过统一出售流入特殊收购势力，武器/宝石/技能书走系统回收。
- 世界物资使用懒结算；导航、位置、地图、出售、自动出售、商场出售、虫洞入口、虫洞奖励和修仙早报会补算相关地点或势力状态。

- 背包物品有分类、品级、重量、堆叠上限、跑商状态、可使用状态、基准价、效果和说明。

## 六、经济系统

货币：

- 默认货币显示名为原石；世界皮肤包可以替换显示名。
- 随身货币可直接消费。
- 银行货币需要取出后才能消费。
- 内部字段统一为 `players.raw_stones`；`bank_accounts` 只表示银行组件，不再作为货币命名依据。

主要收入：

- 签到。
- 探险经验和掉落。
- 普通跑商利润。
- 特殊战利品出售。
- 二手市场出售。
- 武器、宝石、技能书通过商场统一出售。
- 虫洞和首领奖励。
- 福袋。

主要支出：

- 银行升级。
- 跑商买货和手续费。
- 二手市场购买。
- 装备升级。
- 武器升级。
- 押注决斗。

经济保护：

- 普通跑商原地 4 小时转售限制。
- 普通跑商出售价格按最近买入价封顶，最高参考利润 45%。
- 普通跑商出售利润使用柔性收益曲线，活跃人数抬高全服收益线，个人和全服出货越多后续利润越薄，但不禁止继续买卖。
- 跑商买入手续费 1%，出售手续费 2%。
- 二手市场手续费 5%。
- 战利品出售、武器出售、宝石出售、技能书出售都使用柔性降价曲线。
- 柔性降价不禁止玩家继续赚钱，只降低当天后续收益倍率。

## 十、宗门基础

命令：

```text
宗门
宗门成员 [宗门名]
建立宗门 x y 宗门名
加入宗门 宗门名
退出宗门
确认退出宗门
取消退出宗门
宗门大会
领取宗门大会奖励
```

规则：

- `建立宗门 x y 宗门名` 只能选择世界范围内的荒野空坐标。
- `world_locations` 保存系统保留地点，也就是系统自己使用的占用点；系统保留地点和其它宗门山门坐标不能再次建立宗门。
- 创建者自动成为宗主，并写入宗门成员表。
- 一个玩家只能有一个存世创立宗门；原宗门仍存在时，即使已退出，也不能再次建立新宗门。
- 没有宗门的玩家站在宗门山门坐标发送 `宗门`，会看到“这里是宗门”以及宗门资料。
- 山门现场的第一颗业务按钮是 `加入宗门 宗门名`。
- `宗门成员` 是只读名册；无参数时优先查看自己宗门，没有宗门时查看当前位置山门，带 `宗门名` 时可远程查看指定宗门。
- 宗门成员名册按宗主优先、本期贡献从高到低、入门时间从早到晚排序，最多展示前 20 人，并显示称号、等级、身份和本期贡献占比。
- `加入宗门 宗门名` 必须在对应宗门山门坐标发送，远程发送只提示山门坐标。
- 一个玩家当前只能加入或创建一个宗门。
- 周一到周五可以建立、加入或申请退出宗门；周六和周日锁定成员变动。
- `退出宗门` 会进入 30 分钟冷静期，冷静期结束后发送 `确认退出宗门` 才真正退出；申请 24 小时内未确认会过期，需要重新申请。
- `取消退出宗门` 可以取消待确认申请；普通成员确认后离开宗门，宗主确认时优先移交给入门最早成员，没有其它成员且本期宗门大会影响力为 0 时才会解散宗门。
- 宗门大会按周一到周日循环，周一到周六为战斗计分日，周日为本期奖励领取日。
- 加入宗门的成员在战斗计分日抢劫，才会为抢劫发起时所属宗门增加影响力；没加入宗门不计入，周日抢劫也不计入。
- 每个宗门每个周期只保留一条影响力累计记录，下一个周期从新的 `cycle_start` 重新从零开始。
- 宗门大会同时累计个人贡献账，`宗门大会` 面板展示个人贡献值和个人贡献占本宗门影响力的百分比。
- 宗门解散后，历史影响力保留在账上，但本周期失去宗门排行、个人贡献排行和奖励结算资格。
- 宗门山门会按坐标吃城池范围增益；城池等级就是影响半径，城池范围增益先乘宗门等级承载率，再并入最终宗门增益，不生成独立个人属性。
- 多城覆盖按主影响城、第二城 30%、第三城 15% 衰减，第四座及以后只展示覆盖数量，宗门城池增益整体硬上限 75%。
- 周日生成本期宗门大会奖励后即可领取；未领奖励会一直保留，并通过通知栏提醒。
- 宗门大会影响力前 20% 的存世宗门入围；入围宗门按 `30% * (1 + 本宗门最终宗门增益)` 的当前成员比例随机生成淬锋丹待领奖励，向上取整。
- 全服个人贡献前 15% 获得个人贡献奖，每人 1 颗淬锋丹，向上取整；个人奖和宗门随机奖可以同时获得。
- 淬锋丹只从宗门大会奖励获得，进入纳戒，通过纳戒组件的 `武器升限` 消耗，提高武器等级上限，最高 100。
- 当前宗门已开放建宗、入宗、退宗、宗门大会、淬锋丹奖励、山门地点展示、城池范围增益展示、宗门三底蕴、宗门等级和最终宗门增益。
- 宗门不做复杂经营菜单，不新增日常打卡、手动捐献仓库或复杂职位树；宗门成长来自成员抢劫、探险、出售世界物资、挑战首领和挑战虫洞等现有行为自动沉淀。
- 宗门底蕴收为影响力、供养、山门建设三条账；古物、藏宝图、首领、虫洞和战备带来的秘藏与威名表现统一折入影响力。
- 宗门等级上限 100，由三条底蕴自动折算，不设置手动升级按钮；宗门经验 = `山门建设 * 1.0 + 供养 * 0.65 + 影响力 * 0.45`，升下一级经验 = `floor(1500 * 当前等级^1.65 + 3600 * 当前等级)`。
- 宗门满级后继续沉淀三底蕴数值和流水，但宗门等级经验不再保存溢出。
- 宗门最终增益 = `宗门基础增益 + 城池范围增益 * 城池承载率`，最高 80%；宗门基础增益最高 30%，城池承载率由宗门等级决定。
- 宗门增益优先服务宗门大会影响力、宗门随机淬锋丹发放比例、战后休整、普通探险稳定、商路顺药、秘藏线索和公共战斗展示，不能替代武器、装备、宝石和体质成长。
- 宗门相关奖励统一走领取制并通过通知栏提醒，不自动发放。

## 十一、探险系统

命令：

```text
位置
探险列表
探险
探险 地点名
探险状态
结束探险 / 探险结束
探险记录
```

流程：

```text
探险 -> 预计算战斗和掉落 -> 死亡、背包满或到达场次上限时停止 -> 探险状态可提前看 -> 冷却到点后结束探险结算
```

规则：

- `位置` 展示玩家脚下坐标、地貌、功能、可探险信息、商路城池状态、特殊收购/回收建筑或宗门山门信息；如果脚下有散落藏宝图，会展示来源城池、稀品武器和 `领取藏宝图` 入口。
- `探险列表` 展示当前 11 个普通城池和太虚秘境；普通城池会同时展示坐标、推荐等级、怪物等级段、本地三项纯经济特产和城池状态，太虚秘境只展示动态映身和“怪物随进入者变化”，不显示固定推荐等级或映身等级范围；按钮使用 `导航 地点名`。
- `地图` 返回修仙帮助组件的交互地图网页入口，页面展示城池、秘境、特殊收购点、回收建筑、宗门山门和活跃事件，并每 60 秒刷新数据。
- `探险` 不带地点时使用当前位置。
- `探险 地点名` 会先切换到该探险地点再开始。
- 普通探险地点推荐等级只提示风险，不限制进入；太虚秘境没有固定推荐等级，风险由当前玩家等级、随机环境和映身上冲幅度决定。
- 当前探险地点共 12 个；其中 11 个普通探险地点同时是跑商地点和世界状态承接城池，太虚秘境是秘境特色点，不参与跑商，也不承接普通世界物资回收。
- 历史主题已经并入当前 11 个普通地点的地貌叙事和武器池，不再作为独立商路入口或世界状态承接入口。
- 探险占用本体，探险中不能主动挑战首领、虫洞、切磋、押注决斗或抢劫；但正在探险中的玩家可以作为抢劫目标，防守方使用探险开始快照。
- 探险中仍可跑商，解释为行商化身行动。
- 普通探险时长固定 30 分钟。
- 出怪间隔 90 秒。
- 普通探险最多预计算 20 场遭遇。
- 太虚秘境最多预计算 30 场遭遇，最高 45 分钟；打到多少场就按多少场 * 90 秒计算领取冷却。
- 太虚秘境怪物等级以玩家等级为基准浮动，常规为玩家等级 -5 到 +12，小概率 +13 到 +25，极小概率 +26 到 +40；超过 100 级时展示为当前等级显示名加 `秘境强度 +N`。
- 太虚秘境遭遇会生成对应角色“太虚映身”：映身复用玩家当前武器、技能节奏和部分战斗效果，再按秘境环境和等级浮动略微调整；模板怪物只用于命名。
- 太虚秘境每一场都按玩家等级浮动、秘境环境和太虚映身规则独立生成，不使用探险地点固定怪物等级段。
- 太虚秘境领取冷却按实际战斗场数计算，每场 90 秒，并使用专属简化日志。
- 玩家死亡、普通探险 30 分钟到点或秘境达到场次上限时停止预计算。
- 普通探险背包装不下下一件背包掉落时也会提前停止预计算；太虚秘境不掉纯经济特产。
- 结束探险必须等本轮冷却到点后才能结算。
- `探险状态` 可查看当前预计算摘要，包括预计人物经验和预计武器经验。
- `结束探险` 默认返回短结算：战斗胜败、人物经验、武器经验、等级变化、最终状态、最终收获和战斗日志网页链接；逐场摘要、行动次数、技能统计和单场掉落明细进入网页。
- 玩家发送 `战斗日志 开启` 后，结算消息仍保持短摘要，但战斗日志网页默认展开逐次出手；发送 `战斗日志 关闭` 后，网页默认只展开摘要。
- 战败后精神归零。

普通探险掉落结构：

- 背包掉落：药路、民生、建设、古物和战利品；战利品按怪物族群滚自然池，同时保留怪物原本掉落作为偏向。普通探险不掉纯经济特产。
- 纳戒掉落：恢复类、宝石、技能书和低概率流光签。
- 武器掉落：本轮至少胜利过一场后，按整轮概率抽一次，最多 1 把；结算时发放实例武器。

太虚秘境掉落结构：

- 经验：只按秘境少量经验结算，低于普通探险。
- 背包：不掉纯经济特产；秘境会低频掉落古物、战利品和少量高阶建设物资。
- 纳戒：主要掉 1-8 级宝石，直接进入宝石库存，其中 6-8 级为低概率尾奖；另有低概率流光签。
- 武器：每满 3 场进行一次出货判断，也就是第 3、6、9...30 场后判定；武器从全武器池随机，不绑定地点。

怪物经验：

```text
基础经验 = 80 + 怪物等级 * 22
平级倍率 = 1.0
怪物等级高于玩家：每级 +10%，最高 2.0
怪物等级低于玩家：每级 -8%，最低 0.15
最终经验 = 基础经验 * 等级差倍率 * 怪物系数
```

## 十二、战斗系统

战斗原则：

- 攻击力主要来自武器。
- 玩家本体只提供低基础攻击。
- 装备只提供血气、精神、防御等生存属性，不提供攻击力。
- 战斗使用行动条；速度越高，越容易提前出手。
- 当前速度由等级、武器类型、武器攻击负重、命中/闪避类效果共同决定。
- 轻武器速度更快、技能蓄势更勤；重武器单次更重，但速度和技能蓄势更慢。
- 武器自带技能自动释放，不需要单独技能系统。
- 技能书附魔改变武器战斗效果。

伤害公式：

```text
有效防御 = 防御 * (1 - 穿透率)
固定减伤 = min(有效防御 * 0.2, 原始伤害 * 0.5)
中间伤害 = 原始伤害 - 固定减伤
比例减伤 = 有效防御 / (有效防御 + 300 + 30 * sqrt(有效防御))
比例减伤最高 60%
最终伤害 = max(1, int(中间伤害 * (1 - 比例减伤)))
```

战斗日志：

- 探险、虫洞、首领、切磋、决斗默认返回简要摘要。
- 玩家命令 `战斗日志 开启` 后保留详细模式；详细模式让消息里的战斗日志链接默认进入详细视图，逐次出手明细留在网页中展示。
- 玩家命令 `战斗日志 关闭` 后保留简要模式；简要模式消息仍展示关键结算，链接默认进入摘要视图。
- 消息中的链接使用 Markdown 改名显示，例如 `[探险战斗日志〔123〕](完整地址)`，不展示裸 URL。
- 网页路径统一为 `/xiuxian/zhandou-rizhi/...`，玩家侧统一称呼为“战斗日志”。
- 探险战斗日志使用 `exploration_records.record_id`，页面内按第 1 战、第 2 战继续展开；需要直达单场时使用 `#battle-N` 锚点。
- 切磋、决斗战斗日志使用 `duel_records.record_id`；抢劫战斗日志使用 `robbery_records.record_id`。
- 首领战斗日志使用 `boss_challenge_records.record_id`；虫洞战斗日志使用 `wormhole_challenge_records.record_id`。页面总览中继续展示对应事件编号和玩家累计参与数据。
- 战斗日志网页必须展示总览、掉落、结算和逐回合明细；玩家、普通怪、太虚映身、虫洞 Boss 和公共首领挑战里的等级展示使用当前世界皮肤的等级显示名；动态首领和虫洞的详情页不展示具体等级，只显示 `世界平均水平`。

- `修仙信息` 会展示当前战斗日志模式、速度、技能节奏和武器定位；`状态` 会展示其中的关键摘要。
- 战斗日志相关短期明细只保留有限天数；到期清理不影响已经沉淀的长期统计。
- 探险、虫洞、首领、切磋、决斗等涉及人物志或长期称号的记录，到期时需要先汇总到长期统计再清。
- 清理每天最多执行一次。

武器规则：

- 玩家没有武器时会自动补一把初始武器。
- 每把武器 0 级自带 1 个技能，自带技能不占附魔栏。
- 武器类型影响行动速度和自带技能蓄势节奏；高攻击武器还会产生速度负重，避免高攻武器同时成为最快技能流。
- `修仙信息` 和 `查看武器` / `查看武器 武器ID` 会分开展示技能节奏和蓄势基准；蓄势基准越小，技能越容易蓄满。
- 探险武器掉落按整轮判定，不绑定某一场战斗；掉落后随机品质和等级上限，攻击按武器模板基础攻击、品质、等级和武器类型实时派生。武器等级上限按固定权重随机，不受玩家等级影响。
- 武器经验决定武器等级，等级不能超过武器自身等级上限。
- 武器使用独立经验曲线，满级累计经验约 1.17 亿，约为人物 1 到 100 级累计经验的三分之一。
- 武器界面只展示当前等级内的经验进度，例如 `6826/11200`；累计总经验只作为内部等级计算依据，达到自身等级上限后不再累计溢出经验。
- 普通探险、太虚秘境、Boss 和玩家对战都会按本场战斗行动数累计武器经验；人物经验只在本场胜利时按怪物等级和等级差结算，战败不给人物经验。
- 武器升级只受武器自身等级上限限制，不受玩家等级限制；武器等级可以高于人物等级。
- 武器升级只提高等级并解锁附魔栏，攻击数值由实时公式计算，不保存或滚动累加到武器实例。
- `升级武器` 默认升级已装备武器；`升级武器 武器ID` 消耗货币，把指定武器经验补满到下一级所需经验，因此等价于直接提升 1 级。
- 武器升级费用按品质非线性变化：凡品更便宜，良品略贵，珍品更贵，稀品明显更难养成；高品质前期强但培养成本高，低品质高上限武器可以靠便宜升级获得使用价值。
- 武器在 10、25、40、60、80、95 级各解锁 1 个附魔栏，100 级额外解锁 1 个欧皇附魔栏，最多 7 个。
- 附魔栏要求当前等级和等级上限都达到对应门槛。
- 技能书不限制武器类型。
- 附魔不可撤销，附魔后永久占用栏位。
- 同一把武器不能重复附魔同一本技能书；同一流派可以用多本不同技能书组合。
- 已装备武器和最后一把武器不能回收，也不能上架二手市场。

武器和技能书当前规则：

- 自带技能需要明确“打法身份”：高频、多段、爆发、穿透、护身、吸血、压制、持续伤害、斩杀、Boss 协作等至少占一项。
- 技能书附魔不限制武器类型，但要和武器速度系统自然互动：轻武器更适合高频/连击/扰乱，重武器更适合重击/破防/斩杀，杖铃盘盾刃类更适合控制/续航/反击/Boss 协作。
- 技能书按 9 个流派各 4 本配置，共 36 本；可扩到每个流派 5 到 6 本，不追求一次性塞满。
- 每本技能书必须是“主效果 + 清晰代价”，不能只给纯正收益。代价可以是蓄势变慢、精神消耗增加、技能威力降低、普通战斗收益低、Boss 战收益低、PvP 收益低或防御下降。
- 单本技能书最多承载 2 个主效果和 1 个代价；同时加速度、加伤害、加生存的万金油技能书不允许出现。
- 同一流派内部要有分工，不只做数值高低。例如高频流派可分为命中高频、连击高频、削行动条高频、低伤高速；重击流派可分为破防重击、蓄势爆发、斩杀重击、慢速护身。
- 附魔不可撤销，因此技能书文案必须明确告诉玩家它适合什么武器、什么玩法、有什么代价。

武器模板落地口径：

- 每个地点的武器都要有不同手感，不只是名字不同：至少在武器类型、攻击区间、速度负重或自带技能方向上区分。
- 同地点武器可以共享地点意象，但不能共享同一个自带技能；允许做同源变体，例如“青岚剑意”系列，但数值和战斗效果必须不同。
- 轻灵或技巧型偏快、低到中攻击，适合高频和扰乱；均衡或控制型偏稳定，适合压制、护身或续航；重器或稀有型偏慢、高攻击，适合破防、斩杀或首领协作。
- 技能书已按每个流派 4 本落地，形成 36 本基础池；确认战斗表现后再补到每个流派 5 到 6 本，避免一次加太多导致数值失控。
- 如果技能书只改变说明文字、战斗中没有真实效果，不允许加入物品库。
 
回收：

- 武器在铸剑阁回收。
- 技能书在藏经阁回收。
- 回收有单件上限和每日柔性降价曲线。
- 当天回收越多，后续估价越低，最低保留 25%。

## 十六、玩家对战

命令：

```text
切磋 对方名称 / 切磋@对方
接受切磋 发起人名称 / 接受切磋@发起人
拒绝切磋 发起人名称 / 拒绝切磋@发起人
决斗 货币数量 对方名称 / 决斗@对方 货币数量
接受决斗 发起人名称 / 接受决斗@发起人
拒绝决斗 发起人名称 / 拒绝决斗@发起人
抢劫 对方名称 / 抢劫@对方
决斗记录
```

规则：

- `切磋` 是无押注对战。
- `决斗` 必须带正整数货币数量和对方。
- 双方必须空闲。
- 每个玩家同一时间只能有一个未处理对战请求。
- 请求 10 分钟超时。
- `决斗记录` 会先展示当前待处理请求：接收方显示接受/拒绝入口，发起方显示等待对方处理；请求下方再展示最近切磋/决斗记录。
- 决斗发起时冻结发起人押注。
- 接受决斗时扣接受人押注。
- 胜者获得双方押注扣除 3% 手续费后的奖池。
- 接受成功后立即结算，并按接受方当前战斗日志模式返回简要摘要和战斗日志链接；详细模式下网页默认展开逐次出手。
- `抢劫` 只能针对正在探险中的玩家；抢劫方使用当前真实状态，被抢方使用探险开始时的战斗快照。
- 同一轮探险最多被抢劫 2 次，同一个抢劫方不能重复抢同一轮探险。
- 抢劫成功会从对方本轮探险预计算结果里移走战利品并发给抢劫方；没有战利品则只记录战斗。
- 抢劫成功后，被抢方对抢劫方仇恨 +1，报复指数 +20；仇恨表示“我想抢回来”的强度，会在抢劫方的 `修仙信息` 和 `状态` 里以死敌展示。
- 带仇恨反抢成功会触发复仇加成，最多额外拿 5 件战利品，并清空这条仇恨。
- 战败后精神归零。
- `决斗记录` 的历史区只展示最近 10 条；切磋、决斗、抢劫等长期记录需要先汇总再清。

## 十七、异界虫洞

命令：

```text
虫洞
虫洞状态
挑战虫洞
虫洞排行
虫洞奖励
```

出现规则：

- 商场购买、商场出售、导航、战利品出售、战利品自动出售都有概率发现虫洞。
- 同一时间只允许一个开启中的虫洞。
- 虫洞持续 60 分钟。
- 每日虫洞数量动态计算：最低 2 次，基础 2 次，每 4 个活跃玩家 +1，最高 8 次。
- 通知冷却 5 分钟。

发现概率基础值：

```text
导航：1.2%
商场购买：1.8%
商场出售：2.0%
跑商自动出售：3.0%
战利品出售：2.5%
战利品自动出售：3.5%
```

挑战规则：

- 玩家必须空闲、血气大于 0。
- 玩家必须在虫洞出现地点。
- 结算落库前会再次复查本体状态、血气和所在地点，避免本体转入探险或移动后仍被结算。
- 探险中不能挑战虫洞。
- 每次挑战后冷却 3 分钟。
- 虫洞 Boss 血量和内部等级按服务器玩家生态动态生成；详情页只显示 `世界平均水平`，具体等级只作为内部战斗参数，并在战斗日志和历史里按等级显示包展示。
- 战败后精神归零。

奖励：

- 结束后发送 `虫洞奖励` 领取。
- 贡献按个人累计伤害占 Boss 总血量的比例计算，不按参与者之间的伤害占比计算；未击杀只打掉部分血量时，单人也不会显示 100% 贡献。
- 排名按参与者累计伤害排序；虫洞退去也能领取参与奖励，但整体奖励系数低于击杀。
- 原石和经验奖励按贡献、排名和玩家等级计算；珍贵物品按贡献概率判定，武器等级上限仍按固定权重独立随机。
- 可能获得货币、经验、恢复类物品、洗髓液、宝石、技能书、武器。
- 珍贵物品均为随机掉落；贡献和排名只提高概率，不会让低贡献玩家失去基础掉落机会。
- 虫洞不产出开孔器，不产出铭刻之羽。

Boss 池和异界法则：

```text
Boss 名池：千刃界游王、陨炉泰坦、赤瘟炼狱君、缄魂巫皇、苍根不死王、回盾镜魔、断星斩王、星环仲裁者
异界法则池：高频连击、重击破防、持续伤害、压制控制、生存续航、反击护身、斩杀收割、首领协作
```

Boss 名、异界法则展示和真实战斗模板分离；虫洞生成时随机一次异界法则和 `combat_kind`，写入 metadata，同一只虫洞所有挑战保持稳定。战备虫洞由特殊收购势力牵引，Boss 名称走妖、魔、鬼、龙、兽、兵戈等异界谱系，但异界法则和真实战斗模板仍随机固化；`虫洞状态` 和 `虫洞奖励` 都要展示对应异界法则。

命令：


风云榜：

- 按当前业务日统计全服公开榜单。
- 当前包含今日最猛、今日最富、今日商魂、今日最勤、今日欧气和今日活跃。
- 今日最富只展示明面资产总量，不展示玩家内部标识。

修仙早报：

- 每个业务日缓存一份。
- 栏目包括头版人物、坊间传闻、商会风向、城池风向和首领动向。
- 头版人物包含今日最猛、今日最富、今日最勤和今日欧气。
- 早报末尾提供按钮建议：风云榜、修仙早报、修仙界历史、商场推荐、首领。

修仙界历史：

- 作为史书总入口，展示最近世界大事、史册分卷和当前留名方向。
- 记录每日世界事件，当前展示最近 5 天。
- 当天条目会按实时数据刷新；旧日条目保持沉淀结果。
- 记录来源包括岁时情劫、异界虫洞、跑商最高收益、Boss 总伤害、珍稀武器和新增玩家。
- 大事记数据写入 `schema_meta`，不新增专用表。

史册分卷：

- 风云榜是当世榜，只展示当前业务日和当前状态，不承担永久历史。
- 人物史榜展示人物历史纪录，例如首创、最高等级、最高公开资产、最多探险、最多跑商、最高战斗伤害和代表武器。
- 宗门史榜展示宗门历史纪录，例如宗门等级、山门建设、影响力、供养、宗门大会周期霸主和淬锋丹发放结果。
- 城池史榜展示城池历史纪录，例如最高城池等级、最高建设余劲、药路、民生、古物蓄能、战备牵引。
- 战斗名局展示值得入史的经典战斗，例如虫洞首杀、首领击破、最高伤害、最长战斗、残血或越级名局；普通战斗只保留短期流水。
- 商路奇闻展示值得入史的贸易事件，例如最大单次利润、最大物资回收、特殊收购、藏宝图拍卖和城池升级牵引。
- 异界虫洞录展示虫洞首杀、最高难度虫洞、击破次数、活跃虫洞和虫洞掉落名器记录。

史册留名原则：

- 普通事件不入史，普通明细只进入短期流水。
- 短期流水用于早报和最近动态，按清理周期删除。
- 周期归档只留冠军、前几名和摘要。
- 史册留名只保存破纪录、首创、极端名局和特殊事件。
- 同类纪录可以被新纪录覆盖；首创事件和周期霸主摘要长期留存。
- 修仙界历史只反映已经发生的事实，不反向发放奖励，也不改写玩法状态。

人物志：

- 公开查看某个玩家的修仙界档案。
- 输入使用玩家名称，也可以直接@对方；@ 到绑定入口时会自动解析到对方主角色，不要求用户输入长 id。
- 展示称号、等级、常驻地点、代表武器和公开事迹。
- 不展示 client_id、详细资产、背包、纳戒、战斗私密明细。

数据库规则：

- `sql.py` 是 schema 和种子数据中心。
- 当前数据库版本为 `2026062805`；`user_groups` 和 `user_identities` 保存原始入口身份到修仙主角色的映射，`user_group_login_challenges`、`user_group_sessions`、`user_group_bind_codes` 只保存短期用户组后台登录和绑定令牌；`wish_pools`、`wish_prizes`、`wish_draw_records`、`wish_user_vouchers` 保存祈愿奖池、抽取记录和凭证库存；`yuanqi_codes` 保存缘契开启码、绑定角色名、指定剧本、过期时间和使用状态；`dongtian_game_tokens` 保存小游戏启动凭证哈希，`dongtian_rounds` 保存启动凭证哈希、单局凭证哈希、过期时间、消费状态和已签出的兑换码，`dongtian_codes` 保存洞天兑换码、奖励快照、实际发放摘要、领取人和领取系数；`second_hand_records.seller_seen_at` 保存二手成交回执是否已被卖家查看；`player_weapons` 保存累计经验但不保存攻击、自带技能和附魔槽位，`players` 保存休息恢复窗口、数字等级和随身货币字段 `raw_stones`，`player_level_labels` 只保存等级当前展示名；`world_locations`、`trade_locations`、`exploration_locations`、`special_buyers`、`recycle_locations`、`city_world_states` 和 `war_prep_states` 都以 `location_id` 为主键，中文名只作为当前世界皮肤展示快照；`trade_prices`、`trade_heat` 和 `trade_buy_locks` 也以 `location_id` 参与主键，避免换皮后市场状态分裂；`trade_goods` 使用固定特产槽位 ID，`weapon_defs.weapon_type_key`、`monster_defs.kind_key`、`item_defs.effect.world_category_key`、`item_defs.effect.world_subtype_key`、`item_defs.effect.medicine_material_role`、品质稳定键和货币稳定键承接数值规则；`world_skin_active` 保存当前皮肤包，`world_skin_snapshots` 保存换皮回滚快照；`sects` 保存宗门山门坐标、创建者和宗主，`sect_members` 保存宗门成员归属，`sect_stats` 保存宗门等级和三底蕴，`sect_merit_records` 保存宗门长期底蕴流水，`sect_influence_records` 保存宗门大会周期影响力累计，`sect_contribution_records` 保存个人贡献累计，`sect_war_rewards` 保存宗门随机奖和个人贡献奖。
- `city_world_states` 保存 11 个承接城池的药路、民生、建设、古物状态和上次懒结算时间；`world_material_records` 保存世界物资回收流水，并作为称号、人物志、早报和清理汇总依据。
- `treasure_maps` 保存藏宝图拍卖、可拾取、成交和领取状态；`treasure_map_bids` 保存出价流水和货币锁定审计。
- `war_prep_states` 保存特殊收购势力战备状态；`loot_subtype` 只保存 `yao/mo/gui/long/shou/bing` 稳定键，不保存当前战利品展示小类名；`wormholes` 同时承接普通虫洞和战备虫洞，战备信息必须以结构化字段或 metadata 保存。
- `game_logs` 记录关键行为流水，包括创建、改名、签到、礼包、休息、自动用药、银行、导航、跑商、战利品出售、装备、武器、对战、虫洞、首领、铭刻、洞天兑换和物品使用等。
- `player_lifetime_stats` 记录清理前沉淀的长期统计。第一次清理只设置统计起点，不回填历史流水；之后每次清理会先汇总到期明细，再删除明细。
- `player_journals` 不是流水表，而是玩家长期里程碑表；查看 `修仙日记` 时会按当前数据刷新创建、等级、资产、地点、探险、跑商、二手、武器、回收、虫洞、首领、对战、铭刻等摘要。
- `player_titles` 按当前数据自动刷新称号池，并自动佩戴优先级最高的称号；称号没有命令入口，只作为消息头和人物志里的近况窗口。
- `daily_fortunes` 保存每日签到气运；`daily_newspapers` 缓存每日早报；`weapon_legends` 累计武器击杀、Boss、对战和最高伤害。
- `combat_logs` 保留 5 天；战斗日志网页读取的探险、对战、抢劫、首领挑战和虫洞挑战短期明细仍按现有流水保留周期清理；世界物资短流水保留 5 天，长流水保留 7 天；洞天兑换码已兑换或过期超过 7 天后清理，洞天启动凭证和单局凭证只保留有效或短期可审计状态；缘契开启码已使用或已过期超过 7 天后清理；普通明细流水和过期辅助数据保留 7 天；未领取探险、未领取虫洞/首领奖励、未完成藏宝图不会被清理误删。
- 每次清理会先删除已不存在玩家的当前态数据，包括背包、纳戒、银行、固定装备、保险箱、称号、日记、未领奖励和非托管武器；历史流水按保留周期处理。
- 服务关闭时 `数据库备份` 包会备份 sqlite 文件。
- 备份默认保留最近 10 份。

## 二十二、开发检查

常用检查：

```text
修仙架构业务自查
修仙冒烟测试
修仙 ws 触发测试
修仙命令压力测试
```

维护边界：

- 通用 HTTP 数据接口由 `后台接口` 承接；像 `用户组` 这种强绑定单个组件的后台页，可以留在对应组件内，但必须在组件说明中写清路径和鉴权边界。
- 地图展示、排行榜页面、通用后台管理页面属于独立前端或后台功能，必须另建清晰边界。
- 新玩法必须遵守二级包不互相导入原则。
- 物品、技能、地点定义变化后，必须同步更新对应组件说明和本设定。
