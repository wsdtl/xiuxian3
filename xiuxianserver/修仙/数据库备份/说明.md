# 数据库备份包说明

数据库备份包没有玩家命令，也没有 HTTP 路由。它只在服务关闭时自动备份修仙 sqlite 数据库。

## 回调

```text
OnEvent.disconnect(priority=100)
```

## 规则

- 服务关闭时执行 `backup_db()`。
- 备份函数为 `backup_database(database: XiuxianDB)`。
- 备份文件保存在本包目录下，文件名按分钟命名。
- 备份前会先初始化数据库、提交当前连接事务，再使用 sqlite 原生 backup API 复制。
- Windows 下备份连接会显式关闭，避免文件占用。
- 默认只保留最近 10 份备份，旧备份会自动删除。
- 同一分钟已有备份时，会自动追加序号，不覆盖旧文件。
- 启动后 3 分钟内、或距离最近一次自动备份不足 30 分钟时，关闭回调会跳过自动备份。
- 空玩家库不会进入自动备份。
- 备份成功或失败都会通过日志输出。

## 组件关联

- 数据库备份在服务关闭时读取修仙数据库连接，先提交并使用 sqlite backup API 复制文件。
- 备份优先级必须高于数据库关闭；`修仙/__init__.py` 关闭数据库前应先完成备份回调。
- 备份不参与玩家命令、帮助命令、百科知识和业务结算，只为上线运维和事故恢复服务。
- 热重启保护、空库跳过和保留最近 10 份用于避免频繁上传文件时挤掉可恢复备份。

## 维护边界

- 调整保留数量：改 `MAX_BACKUPS`。
- 调整备份目录：改 `backup_database()` 中的 `backup_dir`。
- 如果关闭回调改为倒序执行，保持备份优先级高于数据库关闭优先级即可。
