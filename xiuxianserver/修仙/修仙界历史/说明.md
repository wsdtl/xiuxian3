# 修仙界历史组件说明

修仙界历史组件只负责全服公开信息：风云榜、修仙早报、修仙界历史、史册分卷和人物志。

## 命令

```text
风云榜
修仙早报
修仙界历史
人物史榜
宗门史榜
城池史榜
战斗名局
商路奇闻
异界虫洞录
人物志 玩家名称 / 人物志 [CQ:at,qq=对方ID]
```

## 规则

- 风云榜按当前业务日统计 Boss 伤害、明面资产、跑商、探险、珍稀武器和关键行为，只回答“当世谁最风光”。
- 修仙早报每天缓存一份，栏目包括天地气象、今日最猛、今日最富、今日最勤、今日欧气、坊间传闻、商会风向、城池风向和首领动向。
- 修仙界历史是史书总入口，展示史册目录、当世风向和最近 5 天大事；当天会按实时数据刷新，旧日保持历史文本。
- 人物史榜、宗门史榜和城池史榜是历史纪录榜，只展示首创、最高、最多、连冠、满级这类留名结果。
- 战斗名局、商路奇闻和异界虫洞录是历史事件簿，只展示极端战斗、特殊贸易和虫洞大事。
- 普通流水不永久保存；短期流水用于早报和最近动态，周期归档只留冠军、前几名和摘要，史册留名只保存破纪录、首创、极端名局和特殊事件。
- 同类纪录可以被新纪录覆盖；首创事件和周期霸主摘要可以长期留存；历史组件只反映已经发生的事实，不反向制造奖励。
- 人物志可用玩家名称或 CQ/at 查询；CQ/at 会先按用户组解析到主角色。人物志只展示名称、称号、等级、常驻地点、代表武器、公开事迹和世界物资回收倾向，不展示 client_id、库存明细或详细资产。
- 玩家个人日记在 `玩家` 组件中处理。
- 今日气运只通过 `签到` 生成或查看。
- 武器传奇在 `武器` 组件中处理。
- 称号由公共服务按当前数据自动分配并自动佩戴，不提供手动查看或佩戴命令。
- 称号是玩家消息头和人物志里的近况窗口，来源覆盖当前资产、近期探险、跑商、武器状态、回收、虫洞、首领、对战和铭刻等行为倾向。
- 城池风向只展示 3-5 行摘要，优先读取最高建设城池、药路最紧城池、民生最盛城池、藏宝图动向、古物蓄能和战备待牵引势力。
- 系统消息队列只放全服/世界事实，默认展示队首 3 条；战备虫洞、普通虫洞、岁时情劫、藏宝图拍卖将结和散落藏宝图都属于系统消息，队列项使用无框命令链接进入对应玩法或状态页。
- 个人通知栏只放需要当前玩家处理或会错过的事项：休息/探险可结束、奖励待领、对战请求、藏宝图可领取、宗主收到藏宝图、自己竞拍的藏宝图将结束、二手成交到账，以及新手礼包、银行结息、签到等低优先级日常；通知项使用无框命令链接，点击后自动发送对应命令。
- 药路短缺、建设进度、民生高低、神秘蓄能未满不进系统消息队列和个人通知栏，只进入地点文本或修仙早报。

## 组件关联

- 修仙界历史读取全服公开流水和当前状态，汇总探险、跑商、回收、虫洞、首领、对战、珍稀武器、城池风向和世界事件。
- 史册分卷优先读取现有长期统计、当前状态和保留期内结构化记录；需要长期留名时写入 `schema_meta`，不保存普通明细。
- 人物志读取玩家公开档案、称号、代表武器、常驻地点、长期统计和世界物资回收流水，不展示 client_id、库存明细、银行明细和私密战斗细节。
- 城池风向依赖世界物资懒结算结果；战备、藏宝图、药路、民生和建设只展示摘要，不替代地图/地点文本。
- 系统消息队列提醒短期全服事件，个人通知栏提醒待处理或会错过的个人事项；这些短提醒使用无框命令链接，普通世界状态变化沉淀到早报和历史。

## 维护边界

- 需要按钮时统一使用 `修仙/markdown_utils.py`，例如 `button("修仙信息")`。
- 调整榜单和早报文案：改 `XiuxianHistoryService._build_newspaper()` 和对应排行函数。
- 调整大事记内容：改 `XiuxianHistoryService._build_chronicle_day()`。
- 调整史册分卷：改 `XiuxianHistoryService.history_volume()` 和对应 `_history_*_lines()`。
- 调整人物志展示：改 `XiuxianHistoryService.profile()`。
- 大事记存放在 `schema_meta`，不需要新增表。
- 调整气运：改 `common.py` 的 `FORTUNE_POOL`。
- 调整称号规则：改 `common.py` 的 `CoreService.refresh_titles_conn()`。
- 调整城池风向：读取 `city_world_states`、`world_material_records`、`treasure_maps`、`war_prep_states` 和 `wormholes` 的懒结算结果。
