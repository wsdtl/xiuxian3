# 铭刻组件说明

铭刻负责个性化显示名，只改变外观文本，不改变属性、价格、掉落、战斗结算和物品本体。

## 命令

```text
铭刻
铭刻之羽
铭刻装备 装备位 新名字
铭刻武器 武器ID 新名字
铭刻技能 武器ID 新名字
铭刻附魔 武器ID 附魔序号 新名字
```

以上命令末尾都可以指定铭刻之羽编号，例如：`铭刻武器 武器#12 青云剑 #1`；不指定时默认消耗最早的一枚。

统一入口也可以这样写：

```text
铭刻 装备 头部 青云冠
铭刻 武器 武器#12 青云剑
铭刻 技能 武器#12 青云斩
铭刻 附魔 武器#12 1 青云破
```

## 规则

- 每次成功铭刻消耗 1 枚铭刻之羽。
- 铭刻之羽只能由岁时情劫首领产出，不进入普通探险和异界虫洞奖励。
- 铭刻之羽按实例保存，每一枚都有来源首领、标题和文案。
- `铭刻之羽` 可以查看未使用的遗羽和对应文案。
- 铭刻成功后，该枚铭刻之羽会直接删除，来源与故事随之消散。
- 铭刻后的对象只保留新显示名，不保留羽毛故事。
- 新名字最长 12 个字符，不能包含空白字符。
- 装备按装备位定位，例如：头部、护甲。
- 武器按实例 ID 定位，例如：武器#12、#12、12。
- 自带技能按武器 ID 定位，例如：铭刻技能 武器#12 青云斩。
- 附魔按武器 ID 和附魔序号定位，序号在武器列表或 `查看武器` 中查看。
- `铭刻技能` 不写附魔序号时改武器自带技能；写了附魔序号时改对应附魔。
- 也可以在命令末尾指定铭刻之羽编号，例如 `#1`。
- 同一个对象可以重复铭刻，每次覆盖都重新消耗铭刻之羽。
- 展示时保留原名，例如：青云剑（青岚短剑）、青云破（风刃书）。

## 组件关联

- 铭刻之羽只由首领奖励产出，进入独立铭刻实例表；铭刻组件负责查看和消耗。
- 铭刻可作用于装备、武器、自带技能和武器附魔，只改玩家实例显示名，不改全局定义、战斗效果、价格、掉落和回收估值。
- 武器详情、玩家面板、战斗日志、修仙百科的玩家上下文和人物志需要优先展示铭刻名，并保留原名。
- 保险箱、二手市场、贸易服务回收和战斗结算不得把铭刻名当成新的基础物品定义。

## 维护边界

- 铭刻不修改全局定义表，只修改玩家自己的实例显示名。
- 武器自带技能复用 `weapon_enchant_names`，其中 `slot_no=0` 表示自带技能，`slot_no=1..n` 表示附魔槽位。
- 新增可铭刻对象时，优先给目标实例表加 `custom_name`。
- 如果目标没有实例表，再单独建 `目标_id + custom_name` 映射表。
