# 数值统筹规划

这份文档是全服运营数值总账，用来约束成长、经济、公共事件和清理周期。组件说明负责局部规则，本文件负责判断这些规则放在一起是否健康。

## 一、运营定位

- 主动玩法是主要收益来源：探险、跑商、出售、对战、首领、虫洞和藏宝图竞争都应比纯被动收益更有价值。
- 洞天福地是附加游乐场收益，只发少量修仙资源；它不能替代探险、跑商、首领或虫洞这类主线玩法。
- 被动收益只做兜底：签到、银行结息和每日跑商奖励让低活跃玩家有体感，但不能压过主动玩法。
- 公共成长靠多人行为推进：宗门等级、城池等级、药路、民生、战备和古物蓄能都由玩家已有行为自然沉淀，不新增复杂打卡入口。
- 稀有事件要有限频：虫洞、战备虫洞、藏宝图、首领和宗门大会奖励都需要每日、周期或唯一槽限制，避免连续刷屏和奖励泛滥。
- 所有等级到达上限后丢弃溢出经验，不为提高等级上限缓存历史超额收益。

## 二、成长曲线锚点

| 成长线 | 上限 | 1 到满级累计 | 定位 |
| --- | ---: | ---: | --- |
| 人物等级 | 100 | 351398872 | 角色主成长，最长线 |
| 武器等级 | 100 | 117132935 | 约人物三分之一，战斗长期熟练度 |
| 宗门等级 | 100 | 129266969 | 多人公共长期目标 |
| 城池等级 | 107 | 220526841 | 世界公共长期目标，等级等于影响半径 |

说明：

- 武器经验曲线使用人物经验曲线同族公式，但基数约三分之一；武器战斗经验按战斗时长、对战难度、实际出手和承压综合计算，并有不出手保底。
- 宗门经验来自三底蕴：`山门建设 * 1.0 + 供养 * 0.65 + 影响力 * 0.45`，升下一级为 `floor(1500 * 当前等级^1.65 + 3600 * 当前等级)`。
- 城池建设经验来自建设类世界物资回收，升下一级为 `floor(1200 * 当前等级^1.8 + 3000 * 当前等级)`。
- 人物、武器、宗门、城池到达上限后继续收到经验时只保留满级状态，多余经验直接丢弃。

## 三、收益层级

| 层级 | 来源 | 约束 |
| --- | --- | --- |
| 主收入 | 探险掉落、跑商利润、世界物资出售、战利品特殊收购、首领、虫洞、藏宝图 | 受战斗难度、利润曲线、回收曲线、事件频率限制 |
| 活跃补贴 | 每日跑商奖励、签到 | 低额、每日一次或业务日限制 |
| 附加游乐 | 洞天福地兑换码 | 可一直玩，但兑换收益按当日次数缓慢衰减 |
| 离线兜底 | 银行结息 | 受容量、小时利率、每日利息上限和升级成本限制 |
| 公共反馈 | 宗门底蕴、城池状态、药路顺药、民生恩赐、战备虫洞 | 主要改变世界状态和概率，不直接变成无限个人收益 |

每日跑商奖励是普通跑商净利润的 3.5%，上限 `2000 + 玩家等级 * 350`；它只做活跃补贴，不是主收入。

洞天福地兑换按兑换人的“兑换前今日已兑换次数”压制资源收益：资源系数 `max(0.45, 1 / (1 + 0.055 * 已兑换次数))`。恢复类奖励不直接写入兑换码，而是写入药胚；玩家兑换时按今日已成功成药的药息点稳定成药，前 2 点 100% 稳定，之后稳定率为 `max(0.24, 1 / (1 + 0.17 * (今日药息点 - 2)))`。第一枚兑换码不衰减，后续缓慢下降；玩家可以一直玩，但低量经验、少量原石、药胚、流光签和低概率纳戒特殊物不会无限放大。`灵溪垂钓` 先走启动凭证和单局凭证，再按服务端认可分数发码，分数最高按 3600 计入，单码基础原石不超过 760、基础经验不超过 60；370 分正常局约为 `基础原石 +136`、`基础经验 +11` 和 1 个微光药胚。`灵泉十滴` 一局最长 5 分钟，按服务端单局实际时长、清关数、总爆裂、最高连锁、剩余水滴和效率重新裁分，最高按 5000 分计入，单码基础原石不超过 720、基础经验不超过 58；`碎星切玉` 一局 90 秒，最后 30 秒星潮爆发，按玉尘分数、切玉数、特殊玉、最高连斩和漏切数重新裁分，最高按 3600 玉尘计入，单码基础原石不超过 720、基础经验不超过 54；`剑锋插阵` 一局 90 秒，最后 30 秒万剑归宗，前三次撞剑消耗护心，第 4 次撞剑提前结算，按剑意分数、插剑数、破阵数、天隙次数、最高连斩、爆发期插剑数和失误数重新裁分，最高按 3000 剑意计入，单码基础原石不超过 650、基础经验不超过 50。高分只提高低量基础资源和小概率流光签/洗髓液，不发放淬锋丹、开孔器、铭刻之羽。

## 四、消耗和回收

- 长期货币消耗：银行升级、装备升级、武器升级、跑商买货、二手市场购买、藏宝图出价和决斗押注。
- 回收不设硬封口：世界物资、战利品、武器、宝石、技能书都可以卖，但集中出售会吃柔性降价曲线。
- 柔性曲线只压后续利润或回收价，不阻断玩家操作；玩家可以继续出货，只是收益逐步变薄。
- 纯经济货物用于地区差价，药路/民生/建设/古物进入城池状态，战利品进入特殊收购和战备蓄能。
- 世界皮肤包只替换展示名；世界物资、战备、武器、怪物和品质的数值规则必须按稳定键、基础价、品质键和随机权重计算，不能按当前中文名称、分类名或小类名查固定数值。

## 六、清理周期

- 战斗日志和战斗网页明细保留 5 天。
- 世界物资短流水保留 5 天，长流水保留 7 天。
- 洞天兑换码已兑换或过期超过 7 天后清理；启动凭证和单局凭证按过期时间与短期审计周期清理。
- 普通明细流水和过期辅助数据保留 7 天。
- 未领取探险、未领取虫洞/首领奖励、未完成藏宝图不会被清理误删。
- 清理时会顺手收敛封顶等级经验，保证真实库不会长期积累历史溢出。

## 七、调参入口

| 调整对象 | 主要文件 |
| --- | --- |
| 全局常量、上限、银行、跑商奖励、虫洞每日上限 | `constants.py` |
| 人物经验、武器经验、恢复曲线、利润/回收曲线 | `rules.py` |
| 宗门大会、宗门经验、宗门增益 | `sect_war.py` |
| 城池经验、古物蓄能、藏宝图、战备阈值 | `world_materials.py` |
| 洞天福地兑换码、奖励快照和收益衰减 | `洞天福地/service.py`、`洞天福地/lingxi_fishing.py`、`洞天福地/lingquan_ten_drop.py`、`constants.py` |
| 跑商奖励领取、交易和出售实现 | `贸易服务/service.py` |
| 银行结息实现 | `银行/service.py` |

调参原则：

- 改曲线先看总量，再看单次入口，再看日频率。
- 改收益先确认对应消耗是否存在，避免只进不出。
- 改稀有事件先确认唯一槽、每日上限、通知和领取制是否仍然成立。
- 改等级上限时不要追认旧溢出经验；如果需要补偿，用新活动或新入口发放，不读取历史超额经验。
