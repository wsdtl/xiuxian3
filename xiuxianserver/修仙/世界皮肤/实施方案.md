# 世界皮肤组件实施方案

## 目标

世界皮肤包只负责替换本届世界的展示名，让同一套玩法可以换成另一种世界风格。它不改命令、不改稳定 ID、不改数值公式、不改玩家资产归属。

## 权限

- `.env` 使用 `MASTER_NAME=["邓大楠","邓晓楠"]` 维护可切换皮肤的主人名单。
- 读取方式为 `config.MASTER_NAME`，由 `identity.py` 的 `master_names()` 用 `ast.literal_eval` 解析为列表。
- 权限判断只认当前 `players.display_name`，必要时可额外兼容 `client_id`，但默认不让普通玩家切换。
- 查询当前皮肤和可用皮肤包可以开放给所有玩家；切换只允许主人执行，切换命令内部自动完成校验、写入和失败回滚。

## 包结构

推荐每个皮肤包一个目录：

```text
修仙/世界皮肤/packs/
  default/
    manifest.py
    names.py
  douluo/
    manifest.py
    names.py
  perfect_world/
    manifest.py
    names.py
  cardcaptor_sakura/
    manifest.py
    names.py
```

`manifest.py` 记录包名、版本、作者、说明、适配的 `SCHEMA_VERSION`、包格式版本和条目数量；`names.py` 只保存稳定 ID / 稳定键到展示名的映射，不保存运行数据。皮肤包是受限 Python 数据文件，不是可执行插件：只允许给 `PACKAGE` / `NAMES` 赋值字典常量，允许 `#` 注释和尾随逗号，不允许 import、函数调用、推导式、表达式计算或其它代码。切换器必须用 `ast.parse` + `ast.literal_eval` 读取，不能直接 import。帮助地图不写进业务数据；`帮助` 按 QQ Markdown 公网图片格式展示，并按当前皮肤包名匹配静态资源：`/static/map/{包名}.jpeg|jpg|png`，优先 JPEG/JPG，其次 PNG，匹配不到就使用默认地图。

`names.py` 按服主实际编辑视角分组，不按数据库来源分组：

- `places.cities`：11 个普通城池作为上级实体，每个城池下直接维护 `name`、`roles`、`trade_goods` 和 `weapons`；切换时同步 `world_locations`、`trade_locations`、`exploration_locations`、`city_world_states`、`trade_goods`、武器产地和相关展示快照。
- `places.realm`、`places.buyers`、`places.recycles`：太虚秘境、特殊收购点和回收点；切换时同步 `world_locations`、`special_buyers`、`recycle_locations` 等展示名。
- `world_items`：世界物资按 `medicine/life/build/relic/loot` 五大类和稳定小类拆表；纯经济特产已并入 `places.cities.*.trade_goods`，方便按城池整体审名。
- `ring.recovery`、`ring.gems`、`ring.special`：纳戒物品按消耗方式拆分；开孔器、洗髓液、淬锋丹在 `ring.special`，铭刻之羽不进包。
- `places.cities.*.weapons`、`weapons.skill_books`、`weapons.types`：城池武器模板、自带技能、技能书和武器类型放在同一套结构里；每个武器模板内直接维护武器 `name` 和 `innate_skill.name`，`innate_skill.skill_id` 只作为稳定关联键，不允许编辑；技能书是单一来源，普通版和极版都写在 `weapons.skill_books`，切换时同步 `ring_item_defs.name` 和 `weapon_enchants.name`，不再单独维护重复的“技能书附魔”包段。
- `actors`：怪物、体质、敌方类型、公共敌方技能。
- `wormhole` 和 `secret_realm`：虫洞 Boss、虫洞法则、战备 Boss、战备词缀和太虚秘境环境。
- `system`：品质、货币、等级显示。

当前 `package_format = 4`，每个内置包 `entry_count = 653`：这是合并重复编辑面后的数量，不是漏项。旧口径按数据库来源和代码常量累计 757 个映射项，v4 合并为 653 个编辑项；32 个地点重复项统一到 `places`，72 个技能书/附魔重复项统一到 `weapons.skill_books`，72 个武器自带技能嵌入对应 `places.cities.*.weapons`。

当前内置皮肤包：

- `default`：当前修仙界展示名镜像，用作基准包和回滚校验包。
- `douluo`：斗罗大陆正式风格包，围绕武魂、魂环、魂兽、魂导器、史莱克、武魂殿、七宝琉璃宗、昊天宗和海神岛组织展示名；粒度不足处用魂导与斗魂设定缝合。
- `perfect_world`：完美世界正式风格包，围绕大荒、石村、虚神界、补天阁、百断山、十凶宝术、至尊骨、帝关和仙古组织展示名；粒度不足处用骨文与宝具设定缝合。
- `cardcaptor_sakura`：百变小樱正式风格包，围绕库洛牌、小樱牌、透明牌、友枝町、星杖、库洛书和守护者组织展示名；粒度不足处用魔法阵与牌灵设定缝合。

## 可替换范围

以 `世界皮肤包边界清单.md` 为准，核心包含：

- NPC 地点、11 个普通城池、太虚秘境环境名、地形标签。
- 纯经济特产、世界物资、战利品、恢复药、宝石。
- 怪物名、敌方类型名、公共敌方技能名。
- 虫洞 Boss、虫洞法则显示名、战备 Boss、战备词缀。
- 武器、武器技能、技能书、附魔名、武器类型标签。
- 体质名、体质阶位、体质流派标签。
- 品质显示名：默认凡品、良品、珍品、稀品；内部 `quality_*` 稳定键和数值不跟随改名。
- 货币显示名：默认原石；内部 `raw_stones` 稳定键、字段和公式不跟随改名。
- 等级显示名：默认 `LV1` 到 `LV100`；用于玩家、普通怪、普通探险推荐、虫洞 Boss 日志/历史和公共战斗日志里的等级展示；动态生成的首领和虫洞详情页不展示具体等级，只显示 `世界平均水平`；内部 `players.level` 数字等级、经验曲线、上限和公式不跟随改名。
- 特殊纳戒道具显示名：开孔器、洗髓液、淬锋丹；内部 `kaikongqi`、`xisuiye`、`cuifengdan` 稳定 ID、消耗入口、掉落来源和数值效果不跟随改名。
- 帮助地图资源：每个皮肤包可以配一张同名地图图；`帮助` 命令按 QQ Markdown 公网图片格式展示，按当前包名匹配 `/static/map/{包名}.jpeg|jpg|png`，优先 JPEG/JPG，其次 PNG，匹配不到就用默认地图。

岁时情劫完全不进入世界皮肤包；它只复用公共战斗核心，文本和技能留在 `首领/seasonal_package.py`。岁时首领详情不展示等级，只显示 `世界平均水平`，首领战斗日志若展示玩家/目标等级，则按公共战斗输出使用通用等级显示名。岁时首领名、称号、地点、故事、专属技能和铭刻之羽文案都不由世界皮肤包覆盖。

## 不可替换范围

- 玩家命令、按钮、组件名。
- `client_id`、`location_id`、`item_id`、`ring_item_id`、`weapon_def_id`、`skill_id`、`enchant_id`、`monster_id`、`raw_stones`、`combat_profile`、`combat_kind` 等稳定键。
- 表名、字段名、数值公式、掉落概率、玩家自定义名、宗门名、铭刻自定义名、玩家称号。
- 铭刻之羽固定不跟随世界皮肤；它走 `inscription_feathers` 实例表和岁时情劫文案，不属于 `ring_item_defs`。开孔器、洗髓液、淬锋丹属于特殊纳戒道具，展示名进包，但必须只按稳定 `ring_item_id` 参与业务。

## 数据写入策略

切换时只更新定义表和当前展示快照：

- `world_locations.name/category/terrain/desc`
- `trade_locations.name/specialties`
- `exploration_locations.name/desc`
- `special_buyers.buyer_name`
- `recycle_locations.name/desc`
- `city_world_states.location_name`
- `war_prep_states.buyer_name/prep_name`；`loot_subtype` 只保存 `yao/mo/gui/long/shou/bing` 稳定键，不写皮肤名。
- `trade_goods.home_location`
- `item_defs.name/category/desc/effect` 中的展示名字段；`effect.world_category_key`、`effect.world_subtype_key` 和 `quality` 只保存稳定键，不写皮肤名。
- `ring_item_defs.name/category/desc`；`category_key` 和 `quality` 只保存稳定键，不写皮肤名。
- `monster_defs.name/kind`；`kind_key` 只保存稳定规则键，不写皮肤名。
- `weapon_defs.name/drop_location/weapon_type`；`weapon_type_key` 只保存稳定规则键，不写皮肤名。
- `weapon_skill_defs.name/effect_desc`
- `weapon_enchants.name`
- `physique_defs.name/grade/kind/desc`
- `quality_labels.label/desc`；`quality_key` 只保存 `quality_common`、`quality_good`、`quality_rare`、`quality_epic` 稳定键。
- `currency_labels.label/desc`；`currency_key` 只保存 `raw_stones` 稳定键。
- `player_level_labels.label/desc`；`level` 只保存 1-100 的数字等级。
- 当前交互快照只允许同步白名单内展示名：`players.location_name`、未领取的 `exploration_records.location_name`、开启中的 `wormholes.location_name/boss_name/result`、拍卖中/可拾取/宗主待领的 `treasure_maps.city_name/weapon_name`，以及 `city_world_states`、`war_prep_states`、`trade_goods` 这类当前状态或定义快照。
- 系统消息队列不单独落库改名，只读取当前事件快照；个人通知栏只展示固定待办入口，不保存也不展示世界皮肤实体名，不参与世界皮肤切换。
- 玩家名、宗门名、玩家称号、铭刻自定义名、伤害、排名、贡献、原石、经验、数量和时间这类运行事实不重写。
- `world_skin_active` 记录当前包名、版本、切换人和切换时间；帮助返回内容读取当前包名匹配地图资源。

历史榜、战斗日志、早报、跑商记录、回收记录、世界物资记录和其它已经发生的旧文本/旧流水不作为换皮兼容输入，不按稳定 ID 追改；后续新记录会按当前定义表和公共展示函数写入当前皮肤名。缺少稳定 ID 的旧短期记录可丢弃或保持原文。

## 校验流程

切换前必须先 dry-run：

1. 检查包版本和当前 `SCHEMA_VERSION` 匹配。
2. 检查 `manifest.py` 与 `names.py` 都能按受限 Python 数据文件解析，且 `package_format = 4`；解析时只允许字典常量赋值，禁止执行代码。
3. 检查所有必填稳定 ID / 稳定键都有映射，且同一路径没有重复键。
4. 检查 `places` 下 21 个系统地点完整，其中 11 个普通城池必须都在 `places.cities`，并且每个城池带 `trade_goods` 和 `weapons`。
5. 检查纯经济特产仍为 11 个地区、每地 3 个，且全部挂在对应 `places.cities.*.trade_goods`。
6. 检查恢复药、技能书、武器、宝石、体质和特殊纳戒道具这些强绑定组完整改名。
7. 检查 `places.cities.*.weapons` 完整覆盖 72 个武器模板，且每个 `innate_skill.skill_id` 与数据库 `weapon_defs.skill_id` 一致；检查 `weapons.skill_books` 完整覆盖普通技能书和极版技能书，并能同步到纳戒技能书和附魔定义。
8. 检查品质显示名 4 项完整，且所有 `quality` 字段仍是 `quality_common/good/rare/epic` 稳定键。
9. 检查货币显示名至少包含 `raw_stones`，等级显示名完整覆盖 1-100。
10. 检查 `weapon_type_key`、`kind_key`、`medicine_material_role`、`world_category_key`、`world_subtype_key`、`ring_item_defs.category_key` 完整存在；武器公式、怪物技能、怪物战利品、药路补给、世界物资数值和纳戒物品分流不能依赖当前中文展示名。
11. 检查称号规则只读取统计键、稳定分类和品质稳定键，不依赖当前物品展示名。
12. 检查如果皮肤包提供了同名帮助地图，最终 URL 能由项目公开域名拼出；没有提供时允许走默认地图。
13. 检查岁时情劫没有被包覆盖。
14. 检查写库范围只命中定义表和当前交互快照白名单，不能改历史榜、战斗日志、早报、跑商/回收/世界物资记录等旧记录；个人通知栏不参与切换，系统消息队列只读当前事件快照。
15. 检查切换后 `PRAGMA integrity_check` 为 `ok`。

## 切换流程

命令：

```text
世界皮肤
世界皮肤切换 包名
```

执行顺序：

1. 主人权限校验。
2. 读取包并 dry-run；校验失败时不写库，直接返回失败项。
3. 备份当前皮肤快照到 `world_skin_snapshots`。
4. 单事务写入所有展示名。
5. 同步位置、城市、战备、特产、未过期虫洞、藏宝图等当前展示快照。
6. 跳过历史榜、战斗日志、早报、跑商/回收/世界物资记录等旧记录；个人通知栏不参与切换，系统消息队列从已同步的当前事件快照生成。
7. 事务内执行完整性检查；任一步失败就回滚本次事务，不留下半切换状态。
8. 记录 `world_skin_active` 当前包名、版本、切换人和切换时间。
9. 返回短报告：改了哪些组、同步了哪些当前事件快照、是否通过完整性检查。

## 运行态缓存

- 皮肤包读取由 `world_skin.py` 按 `skin_id + manifest.py 修改时间 + names.py 修改时间` 缓存；修改包文件后会自然重新解析。
- 当前世界名录由 `world_skin_active` 生成 token：数据库路径、当前包名、快照 ID 和切换时间共同决定缓存版本；同一 token 内帮助、百科和物品查询复用同一份名录。
- 名录缓存同时建立展示名、稳定 ID、类型 + 展示名、类型 + 稳定 ID 四类索引，避免百科和定义查询反复全表扫描。
- 定义资料按同一 token 和 `kind/stable_id` 缓存；返回给业务时复制一份 `row`，避免调用方改动污染缓存。
- `世界皮肤切换 包名` 和数据库启动重放当前皮肤后清理运行态名录与定义缓存；包文件缓存不依赖切换事件，而是由文件修改时间失效。

## 帮助类适配

- `指南`、按钮和命令词固定不换皮，只作为玩家入口。
- `帮助` Markdown 入口读取当前皮肤地图；没有激活皮肤状态时使用默认地图。
- `修仙帮助` 图片仍是命令速查图，不放随皮肤变化的世界名。
- 帮助网页启动时缓存 Markdown，同时读取当前世界名录，首页提供“当前世界”栏目，并把当前地点、物品、武器、怪物、体质和品质加入搜索索引。
- `修仙百科` 把同一批当前世界名录作为结构化资料；具体实体问题优先按当前皮肤展示名回答，命令建议仍使用固定命令。
- Markdown 文档可以写默认皮肤示例，但正式实体名以数据库和当前皮肤名录为准。

## 回滚策略

不提供单独玩家命令回滚。`世界皮肤切换 包名` 失败时自动回滚本次事务，不回滚玩家资产和业务进度。若 dry-run 或完整性检查失败，事务直接中止。
