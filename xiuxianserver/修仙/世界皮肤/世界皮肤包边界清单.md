# 世界皮肤包边界清单

口径：`进包` 表示换世界风格时允许热替换的展示名；`不进包` 表示系统命令、稳定键、玩家自定义内容或岁时情劫内置内容，不跟随世界皮肤包自动改名。切换时只同步定义表和当前交互快照；系统消息队列读取当前事件快照，个人通知栏只展示固定待办入口，不参与切换；历史榜、战斗日志、早报、跑商/回收/世界物资记录等旧记录不作为兼容输入，也不按稳定 ID 追改。玩家名、宗门名、铭刻自定义名和称号不重写。

皮肤包使用 `manifest.py` 和 `names.py`。它们是受限 Python 数据文件，不是可执行插件；只允许给 `PACKAGE` / `NAMES` 赋值字典常量，允许 `#` 注释和尾随逗号，不允许 import、函数调用、推导式、表达式计算或其它代码。分组按“实际会一起改名的业务对象”组织，不按数据库表来源组织。当前 v4 包共有 653 个编辑项：旧口径按数据库来源和代码常量累计 757 个映射项，v4 合并了 32 个地点重复项和 72 个技能书/附魔重复项，并把 11 个普通城池整理成 `places.cities` 上级实体；切换器负责把一个编辑项同步回多张定义表。正式读取必须用 `ast.parse` + `ast.literal_eval`，不能直接 import。

### 10. 强绑定批量改名组

原则：以下名字虽然进包，但不能只改单个显示名。换皮时必须按组批量改，连同说明文案、掉落来源、百科问答、通知文本和后续新记录展示口径一起审；已经生成的历史记录不追改。

- 帮助地图资源：每个皮肤包可以提供同名地图图；`帮助` 命令按 QQ Markdown 公网图片格式展示，并按当前包名匹配 `/static/map/{包名}.jpeg|jpg|png`，优先 JPEG/JPG，其次 PNG，匹配不到就使用默认地图。
- 药路恢复组：血契丹、阴冥草、回春露、凝神露、生骨丹、养魂丹；同时绑定药路小类、药路物资、恢复类纳戒物品、商路顺药补给、百科药品说明。

## 三、需要你确认的边界

- `太虚秘境`：当前是特殊战斗模式名，建议固定；但 `幽冥风、镜天影、龙骨尘、星火雨、归墟潮` 这类秘境环境名可以进包。
- 恢复药名和体质名都已经归入进包范围；其中恢复药走“药路恢复组”，体质走“修炼体质组”，换皮时按组批量改。
- 宝石和技能书：显示名适合进包；效果 id、附魔 id 和战斗数值不进包。
- 特殊收购点和回收点：名字适合进包；它们的收购物品类型、价格系数、坐标占位规则不进包。
- 地点显示名：适合进包；当前代码已经把 NPC 地点、特殊收购点、回收点、太虚秘境和城池运行状态拆成稳定 `location_id` 与展示名。后续热插拔时只改展示名和说明文案，不改稳定 ID、坐标占位和玩家自建宗门名。
- 称号：玩家称号本身不进世界皮肤包；称号更新只能读取统计键、稳定分类和品质稳定键，不能依赖当前物品展示名。
- 历史记录：旧短文本和旧流水不追求跨皮肤重写；缺少稳定 ID 的旧短期记录可丢弃或保持原文，新记录按当前皮肤名写入。
- 皮肤切换权限：只允许 `.env` 中 `MASTER_NAME` 对应的玩家显示名执行；普通玩家可查看当前皮肤，但不能切换或回滚；命令入口通过 `Depends(current_is_master)` 注入权限结果，切换命令内部自动校验，失败不写库。
