# 世界皮肤组件说明

世界皮肤用于把本届世界的展示名从稳定业务规则里拆出来。它只替换可展示名字，不改变命令、稳定 ID、数据库结构、玩家资产和数值公式。

皮肤包使用 `manifest.py` 和 `names.py`。它们是受限 Python 数据文件，不是可执行插件：只允许把字典常量赋值给 `PACKAGE` 和 `NAMES`，允许 `#` 注释、尾随逗号和普通嵌套结构，不允许 import、函数调用、推导式、表达式计算或其它代码。`PACKAGE` 只保留加载和展示必需的包身份证字段，具体映射和数量覆盖都以 `names.py` 与切换校验为准。正式切换器只能用 `ast.parse` + `ast.literal_eval` 读取，不能直接 import。当前包格式为 `package_format = 5`，按编辑语义分组，不按数据库表来源分组；v5 要求体质使用结构化记录维护 `name`、`grade`、`kind` 和 `desc`。

`names.py` 的核心分组：

- `places.cities`：11 个普通城池作为上级实体，每个城池下直接维护 `name`、`terrain`、`roles`、`trade_goods` 和 `weapons`；跑商、探险、城池状态、武器产地、藏宝图和虫洞依附都通过同一个 `location_id` 关联。
- `places.realm`、`places.buyers`、`places.recycles`：太虚秘境、特殊收购点和回收点；每个地点同样维护 `name`、`terrain` 和 `roles`，系统地点仍是单源，不再重复写“商路城池”和“探险地点”。
- `world_items`：世界物资除纯经济特产外的五大类及其小类；纯经济特产已经挂回对应城池。
- `ring.recovery`、`ring.gems`、`ring.special`：恢复物、宝石和特殊纳戒道具；开孔器、洗髓液、淬锋丹、流光签放 `ring.special`。
- `places.cities.*.weapons`、`weapons.skill_books`、`weapons.types`：城池武器模板、自带技能、技能书和武器类型；每个武器模板内直接维护 `name` 和 `innate_skill.name`，`innate_skill.skill_id` 是稳定关联键，不能改；技能书是单一来源，切换时同步纳戒技能书和附魔定义。
- `actors`、`wormhole`、`secret_realm`、`system`：怪物/结构化体质/敌方技能、虫洞、太虚环境、品质/货币/等级。

当前内置正式皮肤包：

- `default`：默认修仙界，作为基准包和回滚校验包。
- `douluo`：斗罗大陆风格，围绕武魂、魂环、魂兽、魂导器、史莱克、武魂殿、七宝琉璃宗、昊天宗和海神岛组织展示名。
- `perfect_world`：完美世界风格，围绕大荒、石村、虚神界、补天阁、百断山、十凶宝术、至尊骨、帝关和仙古组织展示名。
- `cardcaptor_sakura`：百变小樱风格，围绕库洛牌、小樱牌、透明牌、友枝町、星杖、库洛书和守护者组织展示名。

品质也按这个口径处理：皮肤包可以替换“凡品、良品、珍品、稀品”的显示名，当前显示名落在 `quality_labels`；数据库业务字段和规则只认 `quality_common`、`quality_good`、`quality_rare`、`quality_epic` 稳定键。

等级显示名也进入世界皮肤：默认 `LV1` 到 `LV100`，当前显示名落在 `player_level_labels`；它用于玩家、普通怪、普通探险推荐、虫洞 Boss 日志/历史和公共战斗日志里的等级展示；动态生成的首领和虫洞详情页不展示具体等级，只显示 `世界平均水平`；`players.level`、经验曲线、升级上限、战斗公式和签到公式仍只认数字等级。

世界物资按 `world_category_key`、`world_subtype_key`、`medicine_material_role`、`base_price` 和品质稳定键计算流向、补给、回收、城池建设、古物蓄能和战备蓄能。当前中文物品名、分类名、小类名只负责展示和玩家搜索，不能作为数值判断依据。

纳戒物品也按同样口径处理：`ring_item_defs.category` 是当前皮肤展示分类，`category_key` 承接恢复类、宝石、技能书、消耗品和专属道具的业务判断；使用、镶嵌、自动用药、掉落、回收和抢劫估值都只看 `category_key`。技能书在包里不放 `ring`，统一放到 `weapons.skill_books`，普通版和极版一起改，切换器同步纳戒技能书和附魔定义。除铭刻之羽外，开孔器、洗髓液、淬锋丹、流光签这类特殊纳戒道具展示名也进入世界皮肤包；业务仍只认 `kaikongqi`、`xisuiye`、`cuifengdan`、`liuguang_qian`。

武器和怪物也按同样原则拆分：`weapon_type_key` 承接武器公式和战备奖励偏向，`kind_key` 承接怪物技能、速度和掉落池；展示名可以进包，稳定键不能进包。

体质也按同样原则拆分：`physique_id` 承接玩家归属，`level`、`physique_value` 和 `effect` 承接重塑概率与战斗/休息/交易加成；皮肤包只改 `physique_defs.name/grade/kind/desc`，用于玩家面板、体质重塑结果、修仙物品查询、帮助和百科检索。

公共规则函数会把已知旧展示名和当前品质显示名映射回稳定键；未知展示名不会原样进入公式，而是落到默认稳定键，避免皮肤包错名污染数值。

地点地貌属于皮肤包展示字段：每个系统地点必须提供 `terrain`，切换时同步到 `world_locations.terrain`，供地图、位置、状态、导航、帮助网页和百科摘要展示；业务规则仍只认 `location_id`、坐标和稳定 key，不按地貌中文名分支。

交互地图使用统一的 `static/map/world-map.html`，皮肤包不再让 `帮助` 命令直接插入地图图片；地图中的地点名、地貌、武器和物资展示名来自当前皮肤包，地图网页入口固定为 `/xiuxian/map`。

当前组件承担边界说明、自动校验和主人切换：

- `世界皮肤包边界清单.md`：列出进包和不进包的名字。
- `__init__.py`：提供 `世界皮肤`、`世界皮肤切换 包名或显示名` 命令入口，并通过 `Depends(current_is_master)` 注入主人权限。
- `service.py`：提供查看和切换业务；切换命令会先自动校验皮肤包，权限判断由命令入口依赖注入传入。
- `world_skin.py`：负责受限读取皮肤包、校验映射完整性、写入定义表和同步当前交互快照。

根目录 `world_skin.py` 维护运行态缓存：皮肤包按 `manifest.py` / `names.py` 修改时间缓存，当前世界名录按数据库、当前包名、快照 ID 和切换时间生成 token 后缓存，并为展示名、稳定 ID、类型 + 展示名、类型 + 稳定 ID 建索引；完整定义资料按同一 token 缓存。切换皮肤和数据库启动重放当前皮肤后会清理运行态缓存，包文件缓存仍由文件修改时间自然失效。

切换世界皮肤必须限制为 `.env` 中 `MASTER_NAME` 对应的玩家显示名；普通玩家只能查看当前皮肤和可用皮肤包，不能执行切换。

## 组件关联

- 读取 `sql.py` 中的稳定 ID、定义表和当前展示名快照。
- 与玩家组件关联权限校验，只认 `players.display_name` 是否在 `config.MASTER_NAME` 中；该判断集中在 `identity.py` 的 `current_is_master` 依赖中。
- 与修仙百科、修仙帮助、探险、贸易服务、异界虫洞、武器、装备、纳戒、世界物资和修仙界历史共享展示名口径，但不改变这些组件的命令和数值公式。
- 修仙根目录 `world_skin.py` 提供当前世界名录只读出口，修仙帮助和修仙百科只读这个公共模块；帮助展示当前世界名录索引和交互地图入口，百科用当前展示名回答具体实体问题。
- 异界虫洞和太虚秘境生成新事件时也只从 `world_skin.py` 读取当前展示名：虫洞 Boss、战备 Boss、异界法则、战备词缀和太虚环境名称/描述会随当前皮肤显示；战斗模板、环境倍率、奖励公式和冷却规则仍由稳定 key 与业务数值决定。
- 岁时情劫完全独立，只复用公共战斗核心，不读取世界皮肤包；岁时首领详情不展示等级，只显示 `世界平均水平`，首领战斗日志作为公共战斗输出时才使用通用等级显示名。铭刻之羽固定留在岁时情劫和铭刻实例体系内，不随世界皮肤改名；首领奖励里的通用纳戒物品、宝石、技能书和武器按各自定义表显示当前皮肤名。
- 玩家称号不进世界皮肤包，称号刷新只允许读取统计键、稳定分类和品质稳定键，不能按当前物品展示名判断。
- 当前切换只同步定义表和当前交互快照：玩家当前位置、城池状态、战备状态、特产定义、未领取探险、开启中的虫洞、拍卖中/可拾取/宗主待领的藏宝图。系统消息队列读取这些当前事件快照，因此会自然展示当前皮肤名；个人通知栏只展示固定待办入口，不保存也不展示世界皮肤实体名，不参与切换。历史榜、战斗日志、早报、跑商/回收/世界物资记录等旧记录不重写，新记录自然使用当前皮肤名。玩家名、宗门名、玩家称号、铭刻自定义名和数值事实不重写。

## 命令

`世界皮肤` 面向所有玩家开放，用来查看当前皮肤和可用皮肤包。`世界皮肤切换 包名或显示名` 只允许 `.env` 中 `MASTER_NAME` 对应的玩家显示名执行；切换时既可以输入目录包名，例如 `douluo`，也可以输入 `display_name`，例如 `斗罗大陆`；切换命令内部会自动校验，校验失败不写库，写入失败自动回滚本次事务。

```text
世界皮肤
世界皮肤切换 包名或显示名
```
