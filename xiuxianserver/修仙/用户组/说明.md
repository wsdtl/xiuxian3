# 用户组组件说明

用户组组件负责把多个通信入口身份绑定到同一个修仙角色。它解决的是“同一个玩家可能从 QQ、WS 或其它适配器进入，但业务上仍然使用同一个角色档案”的问题。

## 概念

- `client_id` 是通信适配器传入的原始入口身份，例如 QQ openid 或 WS 连接 ID。
- `player_id` 是修仙业务使用的稳定角色 ID，当前取用户组主身份 `primary_player_id`。
- 主身份来自已经创建角色的入口账号；新绑定入口只映射到这个主身份，不新建角色。
- 普通玩法命令必须通过 `Depends(current_player_id)` 把 `client_id` 解析成 `player_id` 后再执行业务。
- `创建用户`、`绑定用户组` 这类身份入口必须使用原始 `client_id`，不能提前解析成 `player_id`。

## 命令

```text
用户组
用户组后台
用户组后台登录 登录码
绑定用户组 绑定码
```

`用户组` / `用户组后台` 返回当前环境的用户组后台网页登录地址和完整绑定流程。

`用户组后台登录 登录码` 由已有角色账号发送，用来确认后台页面生成的登录挑战。

`绑定用户组 绑定码` 由尚未创建角色的新入口发送，用来把当前入口绑定到用户组后台登录身份。

## HTTP

```text
GET  /xiuxian/user-groups
POST /xiuxian/user-groups/api/login-challenge
GET  /xiuxian/user-groups/api/login-status
POST /xiuxian/user-groups/api/bind-code
GET  /xiuxian/user-groups/api/identities
```

后台页面先生成登录码，玩家在消息入口发送 `用户组后台登录 登录码` 确认后，浏览器会得到一个短期 session。登录后可以生成一次性绑定码，并查看当前用户组已绑定入口。

玩家找不到网页时，直接发送 `用户组`，返回文本会按当前公网域名、端口和 SSL 状态生成后台地址。

## 规则

- 登录码有效期 15 分钟，只能确认一次。
- 后台 session 有效期 15 分钟，过期后需要重新生成登录码。
- 绑定码有效期 10 分钟，只能使用一次。
- 只有已经创建角色的账号可以确认用户组后台登录。
- 过期登录码、过期 session、已使用或过期绑定码会由用户组流程自动清理。
- 当前入口已经创建过角色时，不能绑定到其它用户组；角色合并、删号和迁移属于单独治理工具，不在本组件内暗中处理。
- 当前入口已经绑定过用户组时，重复绑定会直接提示无需重复绑定。
- 用户组绑定只改变入口身份映射，不迁移背包、纳戒、银行、武器、宗门、战斗记录等业务数据。
- 回复层会按 `player_id` 展示玩家头，但实际发送会回到当前消息上下文里的原始 `client_id`。

## 数据表

- `user_groups`：用户组主表，`primary_player_id` 指向玩家主身份。
- `user_identities`：入口身份表，记录每个原始 `client_id` 归属哪个用户组。
- `user_group_login_challenges`：用户组后台登录挑战。
- `user_group_sessions`：用户组后台登录 session。
- `user_group_bind_codes`：一次性绑定码。

## 组件关联

- 玩家组件创建角色成功后会确保主身份进入用户组表。
- 所有普通玩法组件通过 `launch.adapter.Depends` 和 `current_player_id` 读取当前修仙角色。
- `reply.py` 使用当前适配器上下文发送回复，所以业务服务可以按 `player_id` 读取数据，回复仍会发给触发命令的入口。
- 用户组后台 router 只负责登录和绑定流程，不绕过其它组件的业务规则。
- 多适配器注册由 `launch.adapter.MessageHandler` 统一处理；组件内不直接依赖某一个具体适配器。

## 维护边界

- 新增通信适配器时，只要能提供稳定原始 `client_id`，就可以复用用户组映射。
- 删号、合并已有角色、迁移主身份必须另建账号治理工具，不能复用一次性绑定码直接覆盖已有角色。
- 如果要审计个人绑定历史，可以在不改变现有表语义的前提下新增流水表；当前表只保存当前有效关系和短期登录/绑定令牌。
