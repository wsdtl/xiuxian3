# 探险组件说明

探险负责预计算、自动战斗、自动用药、掉落预览和最终领取。普通探险固定 30 分钟；太虚秘境按实际战斗场数计时。真正发奖和扣损失只在 `结束探险` 时发生。

## 命令

```text
位置 / 地图
探险列表
探险
探险 地点名
探险状态
结束探险 / 探险结束
探险记录
```

## 流程

```text
探险 -> 预计算战斗 -> 死亡、背包满或到达场次上限时停止 -> 探险状态可提前查看 -> 冷却到点后结束探险结算
```

## 规则

- `探险` 不带地点时使用玩家当前位置；`探险 地点名` 会先把玩家位置切到该探险地点再开始。
- 开始探险时会把玩家快照写入本轮结果，后续状态页和结算日志按开始时的血气、精神上限等展示，避免领取前后属性变化导致日志不一致。
- 只有探险地点可以开始探险，特殊收购点、回收点不能探险。
- 当前探险地点共 12 个；太虚秘境是纯探险特色点，不参与跑商。
- 地点推荐等级只用于提示风险，不限制进入。
- 普通探险每 90 秒预计算一场遭遇，30 分钟最多 20 场。
- 太虚秘境每场也按 90 秒计时，最多 30 场，最高 45 分钟；领取冷却等于实际预计算场数乘 90 秒，如果第 3 场重伤，就是 4 分 30 秒后可领取。
- 怪物等级按探险地点的等级段生成，高等级地点不会拦你，但更容易死亡。
- 太虚秘境怪物等级以玩家等级为基准浮动，常规为玩家等级 -5 到 +12，小概率 +13 到 +25，极小概率 +26 到 +40；超过 100 级时不突破等级上限，展示为 `Lv.100｜秘境强度 +N`。
- 太虚秘境遭遇会生成对应角色“太虚映身”：映身复用玩家当前武器、技能节奏和部分战斗效果，再按秘境环境和等级浮动略微调整；模板怪物只用于命名。
- 太虚秘境每一场都按玩家等级浮动、秘境环境和太虚映身规则独立生成，不使用探险地点固定怪物等级段。
- 探险占用玩家本体；探险中不能主动挑战首领、虫洞、切磋、押注决斗或抢劫。
- 正在探险中的玩家可以作为抢劫目标，防守方使用探险开始时写入的战斗快照。
- 探险中仍可跑商，这是行商化身在外界行动，不改变本次探险预计算地点。
- 开始探险时会按玩家的 `自动用药` 开关决定是否预消耗纳戒恢复类物品。
- 自动用药只处理恢复类物品，不会消耗宝石、技能书、开孔器或洗髓液。
- 预计算会在玩家死亡、普通探险 30 分钟到点或秘境达到场次上限时停止；普通探险背包装不下下一件背包掉落时，也会提前停止预计算。
- 预计算停止后不会主动推送消息，玩家必须发送 `结束探险` 才能领取。
- `结束探险` 必须等本轮冷却到点才能结算；提前发送只会提示还要等多久。
- `探险状态` 可以提前查看阶段、剩余时间、停止原因、战斗胜负、预计经验、掉落预览、自动用药和下一步操作。
- `结束探险` 默认返回简要流程日志，包含战斗摘要、技能摘要、掉落、经验和最终状态。
- 发送 `战斗日志 开启` 后，`结束探险` 返回详细逐次出手日志；发送 `战斗日志 关闭` 后恢复简要日志。
- 太虚秘境固定使用专属简化日志，不受 `战斗日志` 开关影响，只展示环境、强度、场次摘要、掉落和最终结算。
- 探险战败后精神会归零；死亡只停止本次预计算，不删除角色。
- 背包掉落进入背包，占负重；恢复类、宝石、技能书进入纳戒，不占负重；武器在结算时发放。
- 普通探险的武器掉落按整轮判定一次：本轮至少胜利过一场，才会按整轮概率抽一把备用武器。
- 太虚秘境使用秘境武器规则：每满 3 场进行一次出货判断，也就是第 3、6、9...30 场后判定；武器从全武器池随机，不绑定地点。
- 太虚秘境只给少量经验，不掉背包特产和怪物材料；主要掉落为 1-8 级宝石和秘境武器，宝石直接进入纳戒的宝石库存，其中 6-8 级为低概率尾奖。

## 扩展

- 新增探险地点：改 `exploration_locations` 初始化数据。
- 新增怪物：改 `MONSTER_DEFS`。
- 新增地点特产掉落：先补齐跑商特产，探险会按当前位置读取额外掉落。
- 调整节奏：改 `EXPLORE_MINUTES` 和 `ENCOUNTER_SECONDS`。
- 调整战斗日志：简要摘要改 `combat_log_text.py`，详细逐回合日志改 `ExplorationService._claim_log_block()` 和逐回合格式化函数。
