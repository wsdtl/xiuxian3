# 祈愿组件说明

祈愿负责消耗纳戒特殊物品 `流光签`，并把结果发放到当前项目已有库存体系。祈愿只服务当前修仙项目，不联动第三方插件，不开放复杂兑换商店。

## 命令

```text
祈愿 / 抽奖
十连祈愿 / 十连抽奖
祈愿奖池 / 抽奖奖池
我的凭证 / 我的奖品
祈愿记录 / 抽奖记录
```

## 规则

- 每次 `祈愿` 消耗纳戒中的 `流光签` x1；`十连祈愿` 消耗 `流光签` x10。
- `流光签` 是纳戒特殊物品，由探险低概率掉落，不占背包负重。
- `流光签` 的展示名进入世界皮肤包；稳定 ID 固定为 `liuguang_qian`，业务只认稳定 ID。
- 奖池默认总权重按 10000 设计，主要产出货币、经验、恢复类纳戒物品和世界物资，少量产出凭证。
- 背包奖品会在扣除 `流光签` 前统一检查负重、格子和堆叠上限；放不下时不会消耗 `流光签`。
- 五种凭证为 `流光凭证`、`玄契凭证`、`星命凭证`、`归墟凭证`、`天启凭证`；当前规则只记账，不开放兑换。
- 祈愿演出只支持 `单抽出蓝`、`单抽出紫`、`单抽出金`、`十连出紫`、`十连出金` 五档 GIF；十连没有金档结果时统一使用十连出紫演出。
- 祈愿演出通过 QQ Markdown 嵌入公网静态 GIF，和结算结果合并成同一条富文本消息；GIF 文件必须放在 `/static/wish/` 并能被外网访问。

## 组件关联

- 探险负责产出 `流光签`，结算仍走纳戒掉落口子。
- 纳戒负责保存 `流光签`，但不直接消耗它；消耗入口在祈愿组件。
- 背包、纳戒、玩家经验和货币发放复用根目录公共服务能力，不新增第三方插件联动层。
- 世界皮肤只替换 `流光签` 展示名，不改变祈愿概率、消耗或稳定 ID。
