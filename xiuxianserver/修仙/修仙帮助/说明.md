# 修仙帮助组件说明

修仙帮助组件负责网页帮助站、帮助图片和固定导航入口。

## 命令

```text
帮助
地图
修仙帮助
指南
指南 成长
指南 行囊
指南 战斗
指南 交易
指南 世界
```

## 规则

- `帮助` 返回 Markdown 帮助入口，隐藏帮助网页和交互地图真实链接，不在消息里直接插入地图图片。
- `地图` 返回交互地图网页入口；页面展示城池、太虚秘境、特殊收购点、回收建筑、宗门山门、虫洞和藏宝图事件，数据接口每 60 秒缓存并刷新。
- `修仙帮助` 返回 `修仙帮助/help.jpg` 指令速查图，优先压成小体积 JPEG。
- `指南` 返回方向索引，只放 `指南 成长/行囊/战斗/交易/世界` 五个方向按钮，避免一个按钮页塞满所有关键业务。
- `指南 方向` 返回该方向的固定命令按钮，只放点击后能直接执行的入口；需要用户补参数的命令不放按钮，只在帮助网页或业务提示里说明。
- 方向别名可按语义进入，例如 `指南 玩家` 等同 `指南 成长`，`指南 探险` 等同 `指南 战斗`，`指南 商场` 等同 `指南 交易`。
- 组件启动时递归读取 `修仙` 下所有 `.md` 文件并缓存；网页请求只读取内存缓存，避免频繁扫描文件造成服务器压力。
- 首页 `主要命令` 从各二级组件说明的 `## 命令` 章节生成；每个组件说明应在 `## 命令` 下放一个 `text` 代码块，代码块内每行一条命令。
- 命令行支持精细格式：`命令｜说明｜参数说明`；网页会自动标记 `无需参数`、`固定选项` 或 `需要参数`。
- 首页搜索会按命令、组件文档和设定文档生成独立结果区，避免把所有栏目直接摊开后混在一起过滤。
- 首页会额外展示 `当前世界` 栏目，资料来自当前世界皮肤和数据库定义，包含地点、物品、武器、怪物、体质、品质等展示名。
- 当前世界资料会进入帮助网页搜索索引；命令词、按钮和组件名仍固定，不跟随皮肤包改名。
- 移动端滚动后会收起搜索框，只保留紧凑顶栏；点击 `搜` 可重新展开搜索。
- 文档卡片和详情页会展示 Markdown 来源文件和最后更新时间。
- 修改 Markdown 后需要重启项目，帮助页缓存才会刷新。
- 网页入口为 `/xiuxian/help`，公开地址由服务端运行配置生成。
- 交互地图入口为 `/xiuxian/map`，动态数据接口为 `/xiuxian/map/data`，网页文件为 `static/map/world-map.html`；地图坐标按 `1 坐标 = 24px` 的 100% 缩放基准绘制。
- 首页按组件分组展示文章流，详情页展示完整 Markdown 和右侧目录。
- `后台接口` 不提供帮助接口，web 后台能力由该组件独立承接。

## 组件关联

- 帮助站递归读取 `修仙/` 下稳定 Markdown，组件说明、完整设定、架构、富文本规则、世界物资设定、数值统筹规划和平衡表都会变成玩家可见知识；封版记录、实施方案、绘图参考和一次性整理文档不进入帮助站。
- 帮助站通过修仙根目录 `world_skin.py` 读取当前世界展示名名录；地图页面通过数据库读取当前点位，换皮肤只换皮肤包资料，不改帮助命令，也不让二级包互相导入。
- 首页主要命令从各组件 `说明.md` 的 `## 命令` 代码块生成；无玩家命令的组件只写 `## HTTP` 或 `## 回调`，不会出现在主要命令分组里。
- 指南按钮只做方向索引；具体业务入口、失败提示和预测按钮仍由各组件文本和回复层决定。
- 修改 Markdown 后需要重启项目刷新缓存；百科也会读取同一批文档，所以文档口径必须同时服务帮助和百科。
