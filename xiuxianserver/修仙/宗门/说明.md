# 宗门

宗门负责山门创建、成员加入/退出，以及宗门大会影响力和奖励结算。

## 命令

```text
宗门
宗门成员 [宗门名]
建立宗门 x y 宗门名
加入宗门 宗门名
退出宗门
宗门大会
领取宗门大会奖励
```

## 约束

- 宗门坐标必须在当前世界地图范围内：左下角 `(-100,-100)`，右上角 `(100,100)`，边界点可用。
- `world_locations` 中的 NPC 地点是系统自己使用的占用点，不能建立宗门。
- 已经被其它宗门占用的坐标不能再次建立宗门。
- 一个玩家当前只能加入或创建一个宗门。
- 一个玩家只能有一个存世创立宗门；原宗门仍存在时，即使已退出，也不能再次建立新宗门。
- 宗门名沿用角色名基础规则：2 到 12 个字符，不含空白。
- `宗门` 在玩家站在山门坐标时会展示该宗门，并给出 `加入宗门 宗门名` 按钮。
- 只有站在对应山门坐标时，才能发送 `加入宗门 宗门名` 成功加入。
- 宗主创建宗门时会自动写入宗门成员表。
- `宗门成员` 是只读名册；无参数时优先查看自己宗门，没有宗门时查看当前位置山门，带 `宗门名` 时可远程查看指定宗门。
- 宗门成员名册按宗主优先、本期贡献从高到低、入门时间从早到晚排序，最多展示前 20 人，并显示称号、等级、身份和本期贡献占比。
- 宗门成员可以在周一到周五退出；普通成员直接退出，宗主退出时优先移交给入门最早的成员，若没有成员则解散宗门。
- 宗门山门会按坐标吃城池范围增益：城池等级就是影响半径，山门落入半径内时会在 `宗门` 和 `宗门大会` 面板展示主影响城、协同城和总增益。
- 城池覆盖强度按 `城池等级 * 距离系数` 排序；主影响城完整计入，第二座按 30% 协同，第三座按 15% 协同，最终硬上限 75%；第四座及以后只展示覆盖数量，不计入当前增益。
- 周六和周日是宗门大会结算与领取窗口，不能建立宗门、加入宗门或退出宗门。
- 宗门大会周期按周一到周日计算，周一到周六为战斗计分日，周日为本期奖励领取日。
- 周一到周六，加入宗门的玩家抢劫才会按抢劫发起时的宗门归属累计宗门影响力；没加入宗门不计入。
- 宗门大会抢劫影响力会吃当前宗门最终宗门增益；传入宗门和玩家当前成员关系不一致时不计入，避免跨宗门误记账。
- 周日抢劫只用于领取本期结算奖励，不进入新的宗门大会影响力累计。
- 每个宗门每个周期只保留一条影响力累计记录，下一个周期从新的 `cycle_start` 重新从零开始。
- 宗门大会同时累计个人贡献账，`宗门大会` 面板展示个人贡献值和个人贡献占本宗门影响力的百分比。
- 宗门解散后，历史影响力保留在账上，但本周期失去宗门排行、个人贡献排行和奖励结算资格。
- 本期影响力前 20% 的存世宗门入围；入围宗门按 `30% * (1 + 本宗门最终宗门增益)` 的当前成员比例随机生成淬锋丹待领奖励，向上取整。
- 全服个人贡献前 15% 获得个人贡献奖，每人 1 颗淬锋丹，向上取整；个人奖和宗门随机奖可以同时获得。
- 周日生成宗门大会奖励后即可领取；未领奖励会一直保留，并通过通知栏提醒。
- 淬锋丹只由宗门大会奖励产生，进入纳戒，消耗入口由纳戒组件承接，命令是 `武器升限`。

## 设计边界

- 宗门不是复杂经营菜单，不新增日常打卡、审批流水、职位树、手动捐献仓库或远程管理面板。
- 宗门成长来自成员已经会做的事情：抢劫、探险、出售世界物资、挑战首领、挑战虫洞、参与宗门大会。
- 玩家常用入口保持少命令：`宗门` 看山门和近况，`宗门成员` 看当前名册，`宗门大会` 看周期排名和奖励，`领取宗门大会奖励` 处理待领事项。
- 宗门面板只展示关键状态：宗门等级、宗门底蕴、城池范围增益、当前生效增益、宗门大会排名、待处理奖励。
- 宗门奖励走领取制，不自动发放；通知栏只提醒玩家有待处理事项。
- 未落到数据库和业务代码的能力，玩家文本不能写成已生效。

## 宗门底蕴

宗门底蕴是宗门长期账，不是个人背包材料，也不是玩家手动捐献仓库。成员离开宗门后，已经沉淀到宗门的底蕴不跟人走。

底蕴收为三条：

| 底蕴 | 来源 | 用途方向 |
| --- | --- | --- |
| 影响力 | 宗门大会抢劫、古物、藏宝图、首领、虫洞、战备事件 | 宗门排面、宗门大会排行、公共战斗声望、淬锋丹发放加成 |
| 供养 | 药路、民生、顺药、恢复药流动和跑商补给 | 战后休整、商路顺药、普通探险稳定 |
| 山门建设 | 成员出售建设物资和城池建设相关行为 | 宗门升级主经验、宗门根基、城池增益承载 |

落地口径：

- `影响力` 已落地为宗门大会周期账和个人贡献账。
- 古物、藏宝图、首领、虫洞和战备带来的秘藏与威名表现，统一折入 `影响力`，不单开秘藏/威名两条底蕴。
- `山门建设` 和 `供养` 优先从现有行为流水和世界状态中自动沉淀，不新增“捐献物品”命令。
- 个人贡献可以单独展示，但宗门底蕴本身是宗门账；宗门不能因为个人跳槽而回滚历史底蕴。
- 宗门解散后，当前周期宗门大会结算失去入围资格；长期底蕴只作为历史记录，不提供增益。

## 宗门等级

- 宗门等级上限为 100。
- 山门建设不单独设置等级，只作为三条底蕴之一。
- 宗门等级由三条底蕴自动折算，不设置手动升级按钮。
- 宗门等级决定宗门基础增益，也决定能吃多少城池范围增益。
- 宗门等级只解锁或放大宗门效果，不直接给角色裸属性，也不替代武器、装备、宝石和体质成长。
- 前期等级要给小宗门体感，中后期变成长期宗门目标，不能被单人短时间刷满。
- 宗门面板展示格式：`宗门 Lv.X｜底蕴：影响力/供养/山门建设`。

经验公式：

```text
宗门经验 = 山门建设 * 1.0 + 供养 * 0.65 + 影响力 * 0.45
升下一级经验 = floor(1500 * 当前等级^1.65 + 3600 * 当前等级)
```

- 宗门满级后继续记录三底蕴和行为流水，但等级经验不保存溢出；提高等级上限时，不追认旧溢出经验。
- 1 到 100 级累计宗门等级经验约 1.29 亿。宗门等级是服务器长期公共目标，不应被单个短期入口快速刷满。

阶段目标：

| 阶段 | 定位 |
| --- | --- |
| Lv.1-10 | 小宗门成长期，几天到一两周能看到变化 |
| Lv.11-30 | 活跃宗门稳定成长 |
| Lv.31-60 | 宗门差距开始明显 |
| Lv.61-85 | 服务器长期目标 |
| Lv.86-100 | 顶级宗门排面 |

## 宗门增益

宗门增益必须服务已有玩法，不新造一套复杂战力系统。最终宗门增益由两部分组成：

```text
最终宗门增益 = 宗门基础增益 + 城池范围增益 * 城池承载率
最终宗门增益最高 80%
```

宗门基础增益来自宗门等级，任何位置都有，保证外围宗门能活：

```text
宗门基础增益 = min(30%, 30% * (宗门等级 / 100)^0.82)
```

城池范围增益仍沿用当前城池规则：主影响城完整计入，第二城 30%，第三城 15%，城池范围增益本身最高 75%。宗门等级决定能吃多少城池增益：

```text
城池承载率 = min(100%, 22% + 78% * (宗门等级 / 100)^0.72)
```

参考值：

| 宗门等级 | 基础增益 | 城池承载 |
| ---: | ---: | ---: |
| Lv.1 | 1% | 25% |
| Lv.10 | 4.5% | 37% |
| Lv.20 | 8% | 47% |
| Lv.40 | 14% | 62% |
| Lv.60 | 20% | 75% |
| Lv.80 | 25% | 88% |
| Lv.100 | 30% | 100% |

一句话：宗门等级决定自强，城池决定地利。

优先落地的增益：

| 增益 | 主要底蕴 | 效果边界 |
| --- | --- | --- |
| 宗门大会影响力提升 | 影响力、山门建设 | 使用最终宗门增益提高抢劫产生的宗门大会影响力 |
| 战后休整 | 供养、山门建设 | 固定冷却战斗后的恢复窗口可小幅受益；不能刷新无 CD 或动态 CD 战斗的恢复窗口 |
| 探险稳固 | 山门建设、供养 | 普通探险重伤后损失略缓、自动用药压力略降或恢复药消耗效率略好，不能让探险免死 |
| 商路顺药 | 供养 | 成员跑商结算时顺药概率或上限略受宗门供养影响，但仍受当地药路状态制约 |
| 秘藏声望 | 影响力 | 成员接触藏宝图、古物、稀品武器线索时有轻微展示或概率修正，不直接白送稀品武器 |
| 战事号召 | 影响力 | 首领、虫洞、战备虫洞等公共战斗的宗门展示和贡献文本更明显，奖励只做小幅概率或排名展示修正 |

宗门大会淬锋丹奖励：

- 入围宗门比例固定为前 20%，不吃增益，避免强宗既更容易得分又扩大入围门槛。
- 入围宗门的成员随机发放比例吃本宗门最终宗门增益，公式为：`30% * (1 + 本宗门最终宗门增益)`，向上取整。
- 最终宗门增益最高 80%，所以入围宗门最高约给 54% 当前成员生成淬锋丹待领奖励。
- 个人贡献奖仍固定为全服个人贡献前 15%，每人 1 颗淬锋丹，不吃宗门增益。

数值边界：

- 宗门增益以小幅乘法和概率修正为主，不直接发放原石、经验、武器或恢复药。
- 宗门不直接加角色攻击、防御、血气、精神。
- 单项增益必须从最终宗门增益中取用，不能和体质、装备、宝石无限叠乘。
- 城池范围增益必须先乘宗门承载率，再并入最终宗门增益。

## 职位与治理

- 当前只有 `宗主` 和 `成员`。
- 职位治理保持轻量，只允许宗门公告、宗主改山门宣言、长老协助查看信息这类辅助能力；不做复杂权限树。
- 周六和周日成员锁定规则保持最高优先级：不能建立、加入、退出、移交或做会影响结算名单的操作。
- 宗主退出仍优先移交给入门最早成员；没有成员则解散。
- 宗门名称、山门坐标和创建者限制是基础治理边界，任何宗门玩法都不能绕开。

## 城池关系

- 宗门山门不是城池，不承接普通世界物资回收。
- 宗门山门落入城池影响半径时，才获得城池范围增益。
- 城池等级来自建设物资回收，城池等级就是影响半径。
- 宗门受益只看山门坐标，不看成员当前位置。
- 城池影响不是占地，不阻止玩家在范围内建立宗门，也不让宗门吞并城池。
- 城池状态展示仍依托地图、位置和地点文本；宗门只展示自己受哪些城池影响。

## 通知与奖励

- 宗门相关奖励统一保留为待领取状态，玩家通过对应领取命令确认入手。
- 系统消息队列提醒宗门大会收官日、领取日和成员变动锁定等全服周期事项，使用无框命令链接进入宗门或宗门大会入口。
- 个人通知栏提醒待领奖励、藏宝图归宗主和宗门大会奖励等当前玩家待办，使用无框命令链接直达领取或查看入口。
- 系统消息队列和个人通知栏都不刷普通底蕴增长、普通建设进度或常规成员行为，避免每条消息都变成账本。
- 宗门大会奖励继续是淬锋丹的唯一来源；其它宗门增益不能绕开淬锋丹提升武器上限。

## 组件关联

- 宗门成员关系影响抢劫后的宗门大会影响力、个人贡献、宗门底蕴和宗门增益；没加入宗门的抢劫不计入宗门大会。
- 城池建设等级和范围来自世界物资建设回收，宗门山门落入城池范围后获得城池范围增益；宗门本身不承接普通世界物资回收。
- 供养、山门建设和影响力从玩家已有行为自动沉淀，主要来源包括世界物资回收、顺药、抢劫、首领、虫洞和战备事件，不新增手动捐献命令。
- 宗门基础增益和城池范围增益会被玩家休息、探险、商路顺药、宗门大会影响力等业务读取；单项读取必须使用公共宗门增益入口。
- 宗门大会奖励生成待领淬锋丹，通知栏提醒，纳戒组件负责 `武器升限` 消耗。
- `位置` 会展示山门信息和加入按钮；加入宗门必须站在对应山门坐标。
