# 贸易服务组件说明

贸易服务组件负责交易、出售分流和自由导航。普通城池、探险地点和商贸点已经重合，地点列表看探险组件的 `探险列表`，完整点位看修仙帮助组件的 `地图` 网页。

## 命令

```text
商场行情 商品名
商场购买 商品名 数量
商场出售 商品名 数量
商场推荐
跑商记录
跑商限制
跑商奖励
出售 物品名 数量
自动出售
出售全部 武器
出售全部 宝石
出售全部 技能书
藏宝图
藏宝图 城池名
藏宝图出价 数量
领取藏宝图
导航 地点名 / 去 地点名 / 来 地点名
导航 x y / 去 x y / 来 x y
```

## 职责

- 商场买卖只处理 11 个普通探险/城池重合点的纯经济特产。
- `商场推荐` 只推荐当前能买、能装进背包、卖出后有净利润的普通跑商路线；哪怕商誉已满，只要散商还有正利润，也必须给出买点、卖点和数量。
- `出售 物品名 数量` 是统一单卖入口：纯经济走商场出售，药路/民生/建设/古物走当前或最近普通城池吸收，战利品自动前往对应特殊收购点，武器/宝石/技能书自动前往回收建筑。
- `自动出售` 清空背包里的可流通物品：纯经济、药路、民生、建设、古物、战利品和可兜底甩卖的背包杂物；纳戒资产只提示出售方式，不主动处理。
- `出售全部 武器/宝石/技能书` 是纳戒资产的明确批量回收入口，会自动前往铸剑阁、琢玉楼或藏经阁后结算。
- 导航仍由贸易服务组件提供；它只负责移动到命名地点或指定坐标，不负责展示地点列表。
- 玩家可用入口只以本说明的命令表为准；地点列表由探险组件展示，出售分流统一走 `出售`、`自动出售` 和 `出售全部`。

## 跑商规则

- 跑商商品来自 `item_defs` 中的可交易纯经济特产，买到后进入背包，占格子和负重。
- 当前只保留 11 个跑商/探险/城池重合地点各 3 个纯经济特产；太虚秘境是秘境，不参与跑商。
- 商场购买只能买当前位置出售的特产，不能购买恢复药。
- 纯经济特产按槽位分为重货、契货、轻货三种内部供需类型；每个城池有明显长期偏好，不再所有货物共用一条需求曲线。
- `商场推荐` 不再只按最高理论利润贪心排序；它会先按真实结算曲线试算推荐数量，再优先推荐封顶后净利润率更接近上限、损耗更少的路线，并兼顾单位负重收益、终点当日出售热度、每日行情和路线分散。
- 每日行情按业务日、城池和货类稳定生成，04:00 后自然刷新；同一天同一地点同一货类不会每次查看都乱跳。
- 同一地点买入的同一批货，4 小时内不能原地出售。
- 普通跑商出售价格按最近买入价封顶，最高参考利润为买入价的 45%。
- 普通跑商没有硬性买卖上限；出售会先吃“商誉成交”，再把超额部分拆成“散商成交”。
- 全服商机按近 7 天活跃人数动态生成；个人商誉承载为全服平均份额的 1.5 倍，但真实商誉成交仍同时受全服剩余商机约束，不能把全服总量扩容。
- 商誉成交使用柔性收益曲线，当天个人和全服商誉成交越多，后续利润倍率越低，最低保留 35%；全服商誉已满后仍可继续跑商，后续成交自然进入散商。
- 散商成交从 15% 利润起步，并按玩家当天个人散商成交量递减，最低保留 8%。
- 商誉曲线和散商曲线都作用于 45% 封顶、扣除买卖手续费后的满净利润，不把买入成本一起打折，也不绕过单件利润上限。
- 散商成交只给低收益现金，不计入全服商机、出售热度、每日跑商奖励和跑商净利润榜。
- 纯经济货物价格基准上调，不受旧式全局随机价格波动影响；现在使用城市长期供需、每日行情、有效买卖热度和普通利润收益曲线共同约束。
- 买卖热度真实记录成交件数，但价格只读取折算后的有效热度；异常高热度或长期散商不会继续把全服路线压成无利润。
- 每日 04:00 重置跑商奖励；当天商誉内普通跑商出售达到动态数量门槛或动态净利润门槛后，可领取一次额外原石。
- 每日跑商奖励为普通跑商净利润的 3.5%，上限 `2000 + 玩家等级 * 350`；它只做活跃补贴，不能替代跑商本身利润、探险收益或公共战斗收益。
- 普通跑商净利润只统计商誉成交部分；战利品出售、散商成交和普通买入流水不直接进入每日奖励口径。

## 世界流向

- 药路、民生、建设、古物不进入普通跑商商品池，由 `出售` / `自动出售` 转入承接城池。
- 战利品不进入普通跑商商品池，由 `出售` / `自动出售` 转入对应特殊收购势力，并增加战备蓄能。
- 古物出售推动当前位置或自动选择城池的神秘蓄能；蓄能满会生成该城池藏宝图拍卖。
- 武器、宝石、技能书不属于背包流通货，只有 `出售 物品名 数量` 或 `出售全部 类型` 明确触发时才回收。
- 跑商、命名地点导航和战利品出售可能触发异界虫洞发现；荒野坐标导航只移动位置，不触发虫洞。
- 探险中仍可跑商、导航和出售，解释为行商化身在外行动；探险本体位置和预计算结果不被商场行为改变。

## 组件关联

- 贸易服务是交易、导航、出售分流和藏宝图入口；地点详情归探险的 `位置`，地点列表归 `探险列表`，完整世界点位归 `地图` 网页。
- 背包里的纯经济物资走跑商买卖；药路、民生、建设、古物通过 `出售` / `自动出售` 进入当前位置承接城池；战利品进入特殊收购和战备蓄能。
- 武器、宝石、技能书只有明确 `出售` 或 `出售全部` 才回收；普通 `自动出售` 只清背包并提示纳戒资产怎么卖。
- 跑商结算会读取药路状态触发顺药，可能触发虫洞发现，并写入跑商记录、每日奖励和称号/日记统计。
- 导航会改变玩家当前位置；虫洞、藏宝图拾取、宗门加入、城池状态展示和探险默认地点都依赖当前位置。
- 藏宝图由古物蓄能生成，出价锁定随身原石，领取后产出城池特色武器并进入武器库。

## 维护边界

- 新增普通跑商地点：先确认它也是普通探险地点和城池承接点，再改 `TRADE_LOCATIONS` 写入坐标和特产。
- 新增跑商商品：改 `TRADE_LOCATIONS` 和 `TRADE_ITEM_DEFS`，当前每个普通城池保持 3 个纯经济特产。
- 调整价格特色：改 `TradeService.price()` 相关价格曲线和地点供需逻辑。
- 新增特殊收购：改 `special_buyers` 初始化，配置地点、坐标、收购物品和倍率；玩家仍通过 `出售` / `自动出售` 进入。
- 新增系统回收地点：改 `recycle_locations` 初始化，仅用于武器、宝石、技能书回收入口。
