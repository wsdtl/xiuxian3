# 开发约束

本文记录 `xiuxianserver` 的项目和框架维护边界。约束以当前代码为准；修仙业务组件的命令、回复、文档和玩法边界见 `修仙/开发约束.md`、`修仙/architecture.md`、`修仙/完整设定.md`。

## 一、观察和改动边界

- 排查问题时需要看完整项目：`xiuxianserver`、`xiuxianplugin`、根目录文档和项目配置都可以读取。
- 默认改动范围是 `xiuxianserver/修仙` 业务组件和相关约束文档。
- 默认不修改 `xiuxianplugin`。
- 默认不修改框架层：`xiuxianserver/launch`、`auto`、Adapter、生命周期、动态路由加载、CORS、日志和静态挂载。
- 如果任务确实需要改框架层或插件，先明确改动理由、影响面和验证方式。

## 二、框架边界

- `main.py` 只保留 FastAPI 应用入口和 uvicorn 启动参数。
- `launch/` 是基础框架层，只负责配置、日志、CORS、动态路由加载、生命周期、定时任务、静态挂载和 Adapter 生命周期。
- `launch/adapter/ws/` 是 WebSocket 通信适配器，通过 `AdapterMount()` 接入框架生命周期。
- `auto/` 放导入即注册或初始化的公共模块。
- `static/` 是框架静态资源目录；不存在时由框架创建。
- `修仙/` 是业务根包，由框架加载；组件内部约束不写在本文。

## 三、加载和生命周期

- 运行配置由 `launch/config.py` 读取并实例化为全局 `config`；约束文档不重复维护配置清单。
- `FastAPIIncludeRouter(app)` 按 `config.router` 收集和导入模块。
- 配置为 router 的模块必须满足：目录存在、带 `__init__.py`、可以成功导入、暴露 `router`。
- 任一条件不满足，框架应在启动阶段直接报错，不能静默少加载接口。
- `lifespan` 负责统一启动和关闭：挂载静态目录、挂载 Adapter、启动 Adapter、启动调度器、注册定时任务、执行 `OnEvent.connect`。
- 关闭阶段按顺序执行 `OnEvent.disconnect`、关闭 Adapter、关闭调度器。
- 数据库启动和关闭使用 `OnEvent.connect` / `OnEvent.disconnect`。
- 定时任务使用 `Scheduler._sync(...)` 或 `Scheduler._async(...)`，并设置稳定 `id`。
- 不要在业务组件里直接改 `launch/lifespan.py`。

## 四、插件边界

- `xiuxianplugin` 是客户端转发层，默认不由服务端功能开发修改。
- 插件连接 `/ws/bot/{client_id}`，发送 `code/type/message/request_id`。
- 插件负责把服务端 `text`、`image`、`markdown` 回复转成机器人消息。
- 服务端和插件协议有变动时，先明确兼容范围和验证方式。

## 五、文档和验证

- 根 README 说明整个仓库结构和总入口。
- 本文只写项目和框架维护边界，不写本机绝对路径、一次性整理步骤或重复配置清单。
- 修仙组件规则写在 `修仙/` 内部 Markdown；文档内容和代码冲突时，以代码为准，同时更新对应文档。
- 修改框架、配置或业务入口后，至少在可用项目环境中运行：

```powershell
python -B test\修仙_架构业务自查.py
python -B test\修仙_markdown按钮测试.py
python -B -c "from main import create_app; app=create_app(); print([getattr(r, 'path', '') for r in app.routes])"
```

需要确认 `/static`、`/ws/bot/{client_id}`、数据库启动/关闭和 WebSocket handler 索引时，使用 `TestClient` 或实际启动服务进入 lifespan。
